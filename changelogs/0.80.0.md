# Release 0.80.0 (2026-03-21)

## Added

### Page Object Model (POM) E2E framework (D-134)

Complete declarative test infrastructure replacing imperative terminal scraping:

- **Page objects**: BaseStep, InitWizard, EditWizard, DashboardSession with step objects (StackStep, BuildStep, SourcesStep, AgentsStep, ConfirmStep, SearchModal, DomainStep)
- **Test fixtures**: ProjectBuilder (6 factory methods), CLI runner, InteractivePrompt, DualScopeHelpers
- **Custom matchers**: toHaveConfig, toHaveCompiledAgents, toHaveCompiledAgent, toHaveSkillCopied, toHaveLocalSkills, toHavePlugin, toHaveEjectedTemplate, toHaveSettings
- **Centralized constants**: DIRS, FILES, STEP_TEXT, TIMEOUTS, EXIT_CODES, INTERNAL_DELAYS in `e2e/pages/constants.ts`
- **POM smoke tests**: 3 tests proving the framework end-to-end before migration

### Test coverage expansion (D-136)

142 new tests across 15 files:

- **Unit tests**: plugin-manifest-finder (7), build-step-logic (34), config-writer (47)
- **E2E command tests**: new-agent (10), validate (8), doctor (6), eject (8), import-skill (5), config (5), help (6), search-static (3), uninstall (3)
- 3 tests use `it.fails()` to document real bugs with accurate assertions

### E2E standards documentation

7-document guide in `docs/standards/e2e/`: README, patterns, anti-patterns, assertions, page-objects, test-data, test-structure.

## Changed

### DRY migration across all E2E tests

- Replaced 265 raw exit code integers with `EXIT_CODES.SUCCESS`/`EXIT_CODES.ERROR` across 38 files
- Replaced raw UI strings with `STEP_TEXT` constants in 6 files
- Centralized `agentsPath()` in test-utils.ts (removed duplicates from 9 files)
- Replaced `src/cli/consts.ts` imports in project-builder.ts with `e2e/pages/constants.ts`
- Renamed `navigateToItem()` to `waitForItemVisible()` for clarity
- Extracted `DashboardSession` to its own file

### Test parallelization improvements

- Split `dual-scope-edit-integrity.e2e.test.ts` (321s) into 3 parallel files (~120s each)
- Split `doctor.e2e.test.ts` (127s) into 2 parallel files (lightweight + diagnostics)
- E2E suite: 71 files → 74 files, same 554 tests

## Removed

- `plugin-assertions.ts` — replaced by custom matchers
- Legacy helpers: `navigateInitWizardToCompletion`, `passThroughAllBuildDomains`, standalone `waitForRawText`
- Unused `pressTab()` from BaseStep
- Internal timing constants and delay function from test-utils.ts

## Fixed

- `cleanupDirs` now used in `destroy()` for InitWizard and DashboardSession (temp dirs were leaked)
- `InteractivePrompt` inline delay patterns replaced with shared `delay()` method (10 occurrences)
- `PluginScope` type inconsistency between project-matchers.ts and setup.ts
- Unused `projectDir` field removed from InitWizard constructor
- `source-switching-modes.e2e.test.ts` try-catch fallback replaced with `fileExists` check
- Unused imports cleaned: `createPermissionsFile`, `fileExists`, `renderSkillMd`
