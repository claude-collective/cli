# Release 0.74.12 (2026-03-15)

## Changed

- Matrix resolver: split `isDiscouraged` into `isDiscouraged`, `isIncompatible`, and `hasUnmetRequirements` — each concern now has its own function with clear semantics
- `OptionState` type moved from `category-grid.tsx` to `types/matrix.ts` as a structured `{ status, reason? }` object
- Skill labels in build step now only show on the focused skill (not all skills at once)
- Category section headers use inverted highlight style for focused category
- Non-exclusive categories no longer show a selection counter
- Build step no longer blocks advancement when a required category has no selection

## Removed

- Marketplace label display from wizard header
- "Accept defaults" hotkey (`A`) from build step
- Build step validation footer (validation error for empty required categories)
- `getMarketplaceLabel()` from source-loader

## Fixed

- macOS symlink bug: replaced `path.resolve()` with `fs.realpathSync()` in `local-installer.ts` to correctly resolve `/var` -> `/private/var` symlinks when comparing project and home directories
