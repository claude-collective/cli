# Release 0.96.3 (2026-03-29)

## Fixed

- **config-loader**: empty or zero-key config files now return `null` (treated as "not installed") instead of leaking a confusing empty object through validation
- **source-validator**: skills with `custom: true` in `metadata.yaml` are now routed to `customMetadataValidationSchema` (relaxed category/slug) instead of the strict enum-based schema — the schema existed but was never wired in

## Added

- **Command tests** (~73 new tests across 11 command test files):
  - `help.test.ts` — new file covering `cc help`, `cc help <subcommand>`, and unknown command error
  - `new/agent.test.ts` — purpose/refresh flags, compilation error path, no-install error, list visibility
  - `update.test.ts` — hash mismatch detection, no-match error, local-only skill reporting
  - `validate.test.ts` — malformed plugin.json, cross-reference validation, custom skill schema routing
  - `doctor.test.ts` — missing-skills-installed check, broken agent references
  - `search.test.ts` — `--json` flag output, category filtering, no-results path
  - `build/marketplace.test.ts` — malformed/missing-name plugin.json handling
  - `import/skill.test.ts` — name collision without `--force`, partial-success in `--all` mode
  - `eject.test.ts` — overwrite-without-`--force` error, content preservation
  - `config/index.test.ts` — custom source display
  - `uninstall.test.ts` — global scope handling, global-dir invocation

- **Integration test cleanup**:
  - `assertConfigIntegrity()` helper extracted to `helpers.ts` — replaces duplicate assertions across 3 integration files
  - `wizard-init-compile-pipeline.test.ts` merged into `init-flow.integration.test.ts` (recompile round-trip + agent content tests moved; 4 duplicate tests removed)
  - Global scope test added to `init-flow.integration.test.ts`
  - Console spy moved to `beforeEach` in `compilation-pipeline.test.ts`

## Removed

- `src/cli/lib/__tests__/integration/wizard-init-compile-pipeline.test.ts` — unique tests merged into `init-flow.integration.test.ts`
