# Release 0.56.0 (2026-03-02)

## Added

- **Comprehensive E2E test coverage** — 350+ end-to-end tests across all commands and interactive wizard flows. Covers init wizard (source management, global install, dashboard, toggles, flag combinations), edit wizard (source flag, confirm navigation, global fallback), update (interactive confirm, outdated skills), search (import flow, filtering), build-stack (customize, domain selection), compile (--agent-source, global fallback), validate (source override, relationships), and all command help/error paths.
- **D-63: E2E tests in pre-commit hook** — `.husky/pre-commit` now runs `bun run test:e2e` alongside unit tests before every commit.
- **D-65: Global config detection in init** — `agentsinc init` now detects an existing global installation when no project config exists and prompts the user to either edit the global config or create a new project-level config. `detectGlobalInstallation()` exported from installation module.

## Changed

- **D-37 design doc: "custom" → "mixed"** — Renamed the `installMode: "custom"` value to `"mixed"` in the install mode redesign proposal for clarity (mixed = some skills local, some plugin).
- **D-67 scope broadened** — Task updated from narrow alias removal to broader investigation: skill metadata as the single source of truth for intrinsic skill properties, eliminating redundant central config files.

## Removed

- Obsolete implementation plan files for completed tasks: D-36, D-44, D-46, D-54, D-59, T-12, T-08.
- Legacy tracking files: TODO-final.md, TODO-matrix-decomposition.md, TODO-testing.md.

## Known Issues (documented with `it.fails()`)

- `compile` global fallback uses `process.cwd()` instead of `installation.projectDir`
- `doctor` command doesn't fall back to global config
- `update` interactive confirm hangs after pressing 'y' (Ink `waitUntilExit` not resolved)
- `edit` newly-added local skill not visible in build step alongside original skills
- `validate` doesn't detect duplicate skill IDs across categories
- `search` import receives relative skill path instead of absolute
