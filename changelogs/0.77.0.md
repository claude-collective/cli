# Release 0.77.0 (2026-03-17)

## Added

- **4 new default stacks**: Next.js Supabase Full-Stack, Next.js Turborepo Full-Stack, React Old School, React Hono Full-Stack
- **`web-routing` category**: new category for client-side routing libraries
- **Unsatisfiable requires detection**: skills whose required dependencies conflict with current selections are marked incompatible (e.g. SvelteKit marked red when React is selected, because SvelteKit requires Svelte which conflicts with React)
- **Framework incompatibility markers**: skills with `compatibleWith` declarations that don't match the selected framework show as incompatible with reason (e.g. "only compatible with React, Next.js, Remix")
- **Filter Incompatible toggle** (`F` hotkey): optionally hide incompatible skills in the build step (off by default — all skills visible with markers)
- **`getUnmetRequiredBy` resolver function**: finds selected skills with unmet dependencies on an unselected skill
- **"required by" label for unselected skills**: shows when a selected skill needs this skill and the requirement is unmet (disappears once satisfied)

## Changed

- **Default stacks overhauled** (D-115): renamed 9 stacks with framework-first naming and "Full-Stack" suffix, updated all descriptions to comma-separated core skill lists, reordered React stacks first
- **Stack IDs updated**: `t3-stack` → `nextjs-t3-stack`, `angular-stack` → `angular-modern-fullstack`, `vue-stack` → `vue-modern-fullstack`, `solidjs-stack` → `solidjs-fullstack`, `nuxt-stack` → `nuxt-fullstack`, `remix-stack` → `remix-fullstack`, `sveltekit-stack` → `sveltekit-fullstack`, `astro-stack` → `astro-content-fullstack`
- **TanStack Router recategorized**: moved from `web-framework` to `web-routing` with React compatibility rule
- **Category order bumped**: existing web categories shifted +1 to accommodate `web-routing` at position 3

## Removed

- **Framework-first filtering** (default off): skills are no longer hidden based on framework selection — they show with incompatibility markers instead
- **Auto-select of required skills**: selecting Remix no longer auto-selects React; users see "required by Remix" label and decide themselves
- **Deselection blocking**: skills can be freely deselected even if other selected skills depend on them
- **Section locking**: all category sections are navigable without requiring framework selection first
- **"required by" label on selected skills**: no longer shows when the requirement is already satisfied
