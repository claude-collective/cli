# Release 0.123.0 (2026-04-13)

## Added

- **D-194: Explicit stack grouping** — Stacks now have an optional `group` property. Groups render with headers (React, CLI), sorted by explicit order. Stacks without a group fall under "Other Frameworks". When no stacks have groups, headers are omitted entirely.
- **D-195: Auto-preselect agents from stack** — Selecting a stack in the init wizard now preselects agents from the stack's agent keys, merged with global agent preselections. Replaces the old domain-based `DOMAIN_AGENTS` mapping.
- **D-151: E2E session-level timeout** — `TerminalSession` and `BaseStep` now have a `defaultTimeout` property overridable at construction. Wizard launch options accept `defaultTimeout` to propagate to the session. Timeout constants moved to `TIMEOUTS.SESSION_DEFAULT` and `TIMEOUTS.SESSION_DEFAULT_CI`.
- New E2E test for stack agent preselection (`init-wizard-stack-agents.e2e.test.ts`)

## Fixed

- **D-167: Task IDs in describe blocks** — Already cleaned in prior session; fixed `sourceTempDir = undefined!` reset pattern in `search-interactive.e2e.test.ts`
- Default source E2E test updated to assert stack-derived agents instead of hardcoded `web-developer`

## Changed

- `ResolvedStack` and `SuggestedStack` types now include `group?: string`
- `groupStacks()` rewritten to use `stack.group` instead of skill ID prefix inference; `GROUP_ORDER` controls display priority (React first, then CLI)
- `populateFromStack` in wizard-store derives `selectedAgents` from `Object.keys(stack.agents)`
- `stack-selection.tsx` merges stack-derived agents with global preselections, clears excluded flags on re-activated agents
- Magic timeout constants in `terminal-session.ts` replaced with `TIMEOUTS` from `constants.ts`
