# Release 0.106.0 (2026-04-05)

## Added

- Project tracking in global config — `projects?: string[]` field records which project directories have been initialized, enabling cross-project propagation
- `registerProjectPath()` �� automatically registers project directories in global config during init/edit, filters stale entries where `.claude-src/config.ts` no longer exists
- `deregisterProjectPath()` — removes project from global config on `uninstall --all`, uses `path.resolve` (safe for deleted directories)
- `propagateGlobalChangesToProjects()` — when global skills/agents change, updates `config-types.ts` and `config.ts` in all registered project installations
- `projects` field in generated `ProjectConfig` interface (config-types-writer)
- 4 new E2E tests covering registration, propagation, deregistration, and stale path filtering

## Changed

- `writeScopedConfigs` now registers the project path in global config and triggers propagation when global data changes
- `uninstall --all` deregisters the project from global config's tracked projects list
- `projects` field is stripped from generated config source files (only stored in global config on disk)
