# Release 0.65.0 (2026-03-11)

## Added

- `generateConfigTypesSource` accepts optional `ProjectConfig` to narrow `SkillId`/`AgentName`/`Category`/`Domain` unions to only installed items
- `writeScopedConfigs` exported from `local-installer.ts` for reuse across commands
- `gitlab-ci` added to `SkillSlug` union
- `content-generators.ts` — `renderSkillMd()`, `renderAgentYaml()`, `renderConfigTs()`, `renderCategoriesTs()`, `renderRulesTs()` for test file content generation
- Canonical matrix constants (`EMPTY_MATRIX`, `SINGLE_REACT_MATRIX`, etc.) in `mock-matrices.ts`
- `AGENT_DEFS` registry in `mock-agents.ts` for reusable agent metadata
- Stack templates (`SINGLE_AGENT_STACK_TEMPLATE`, `MULTI_AGENT_STACK_TEMPLATE`) in `mock-stacks.ts`
- R-09b plan for remaining test data consolidation (matrices, agents, stacks)

## Changed

- `edit` command uses `writeScopedConfigs` instead of duplicated inline scope-splitting logic
- `writeProjectConfigTypes` now derives project-specific categories/domains from config via matrix lookup
- Config writer generates `Partial<Record<AgentName, StackAgentConfig>>` for stack variables (not all agents have stack entries)
- Step-agents always shows scope badge `[G]`/`[P]` instead of only when selected
- ~90 test files migrated to canonical `SKILLS.*` registry, `createMockMatrix` spread syntax, and `renderSkillMd()`/`renderAgentYaml()` content generators (R-09)
- `mock-skills.ts` consolidated from inline arrays to canonical registry references
- `mock-data/index.ts` barrel file removed — all imports are now direct
- CLAUDE.md updated with R-09 conventions: canonical SKILLS registry, content generators, spread syntax, export hygiene

## Fixed

- Generated config-types.ts unions now reflect only the items actually installed, not the entire matrix
