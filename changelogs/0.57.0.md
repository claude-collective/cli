# Release 0.57.0 (2026-03-02)

## Added

- **Mixed install mode**: Install mode (`"local"`, `"plugin"`, `"mixed"`) is now derived from per-skill source selections rather than set globally with a toggle
- **Per-skill source selections**: `sourceSelections` field in `ProjectConfig` maps skill IDs to source names, persisted in `config.ts`
- **Mode migrator**: `detectMigrations()` and `executeMigration()` handle per-skill transitions between local and plugin modes during `cc edit`
- **Bulk source hotkeys**: `L` (set all local) and `P` (set all plugin) in the sources step customize view
- **"Local" always available**: `buildSourceRows()` ensures every skill shows a "Local" source option regardless of available sources
- **Config persistence in edit**: `cc edit` now writes wizard results to `config.ts` (previously only `cc init` did this)
- **Install mode override in recompile**: `recompileAgents()` accepts an explicit `installMode` option, used by edit flow
- New integration tests for install mode, config persistence, and migration flows
- E2E tests for local install verification (skill file copy, config output, no archive warnings)

## Changed

- `InstallMode` type extended from `"local" | "plugin"` to `"local" | "plugin" | "mixed"` across installation.ts, config.ts, schemas.ts, and generated type definitions
- Confirm step shows mixed mode with skill counts (e.g., "Mixed (2 local, 3 plugin)")
- `setConfigMetadata()`, `buildAndMergeConfig()`, and `writeConfigFile()` exported from local-installer.ts for use by edit flow
- `compileAgentForPlugin()` accepts `InstallMode` type instead of narrow string literal
- E2E init test updated: confirm step shows "Plugin" (was "Local") since default source selections use marketplace

## Removed

- **Archive mechanism removed**: `archiveLocalSkill()`, `restoreArchivedSkill()`, `hasArchivedSkill()` replaced by `deleteLocalSkill()` — local skills are deleted permanently on source switch, not archived
- `ARCHIVED_SKILLS_DIR_NAME` constant removed from consts.ts
- Global `P` hotkey toggle for install mode removed from wizard (replaced by per-skill source selection)
- "Plugin mode" indicator removed from wizard layout footer
- "Toggle plugin/local install mode" removed from help modal
