# Release 0.74.8 (2026-03-14)

## Fixed

- **Edit command scope bugs (D-92/D-93)**: `edit.tsx` now uses `process.cwd()` for all write operations (config saves, plugin installs, scope migrations, recompilation) instead of `installation.projectDir`. When `detectInstallation()` falls back to the global installation, writes now correctly target the user's working directory instead of `~/`.
- **Edit locked-items check**: `isGlobalDir` check uses `cwd` instead of `projectDir`, so global items are correctly locked when editing from a project directory with only a global installation.
- **Stack selection reset (D-94)**: `selectStack()` now clears all previous selections (domains, skills, agents, bound skills) when changing stacks or choosing "start from scratch".
- **Agent recompiler directory creation**: `ensureDir(globalAgentsDir)` is now called unconditionally before the agent compilation loop, preventing potential missing-directory errors.

## Changed

- **GlobalConfigPrompt removed from init**: `init.tsx` no longer prompts "Edit global installation or create new project installation" when a global config exists. The wizard handles scope via [G]/[P] badges and writes route by scope automatically.
- **Wizard footer simplified (D-96)**: Removed redundant left/right arrow navigation hint from wizard footer, keeping only up/down arrows.
- **`ensureBlankGlobalConfig()` called before scoped writes**: Edit command ensures a blank global config exists before writing project configs that import from it.

## Added

- **211 unit tests for 29 previously untested functions**: Direct tests for internal pure functions across 11 source files (output-validator, matrix-resolver, source-loader, source-fetcher, wizard/utils, hotkeys, schemas, versioning, compiler, local-installer, config-merger, source-validator). Total test count: 3516.
- **P3 refactors for testability**: `setConfigMetadata` refactored from mutation to pure return. `mergeConfigs` extracted from `mergeWithExistingConfig`. `validateMetadataConventions`, `validateSkillFilePairs`, `isSnakeCase` extracted from `validateSource`.
- **E2E test updates**: Updated 7 E2E test files and added `unified-config-view.e2e.test.ts` to align with edit cwd fix and GlobalConfigPrompt removal.
