# Release 0.75.0 (2026-03-16)

## Added

- **web-meta-framework category** — New exclusive category for full-stack meta-frameworks (Next.js, Remix, Nuxt, SvelteKit, Astro), separate from base frameworks
- **api-ai category** — New category for AI SDKs and LLM integration patterns
- **api-baas category** — New category for backend-as-a-service platforms (Supabase, Firebase)
- **Required-by UX (D-39)** — When a meta-framework is selected, its base framework shows a dimmed "(required by X)" label and cannot be deselected. Selecting a meta-framework auto-selects its required base framework.
- **T3 Stack** — Next.js + tRPC + Prisma + Tailwind + NextAuth (9 stacks total, up from 6)
- **SvelteKit Full-Stack** — Svelte 5 + SvelteKit + Hono + Drizzle
- **Astro Content Stack** — Astro content-first with islands architecture + Hono + Drizzle
- **9 new skills in marketplace** — Svelte 5 Runes, SvelteKit, Astro, Auth.js/NextAuth v5, NestJS, Supabase, Vercel AI SDK v6, Clerk, MongoDB/Mongoose
- **Framework rules** — Svelte added to base framework conflicts/alternatives, SvelteKit/Astro added to meta-framework conflicts/alternatives, SvelteKit requires Svelte, MongoDB added to database conflicts

## Changed

- **web-framework category** — Now exclusively for base UI frameworks (React, Vue, Angular, SolidJS, Svelte); meta-frameworks moved to web-meta-framework
- **Framework conflict rules** — Split into base framework conflicts and meta-framework conflicts instead of one monolithic group
- **Next.js skill** — Two separate skills (App Router + Server Actions) merged into one unified skill with progressive disclosure
- **Default rules** — Database ORM alternatives now include MongoDB; new alternative groups for AI SDK, Backend as a Service, and Auth

## Removed

- **web-framework-nextjs-app-router** — Merged into unified web-framework-nextjs
- **web-framework-nextjs-server-actions** — Merged into unified web-framework-nextjs
