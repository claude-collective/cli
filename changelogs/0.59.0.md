# Release 0.59.0 (2026-03-07)

## Added

- `AgentScopeConfig` type: agents now have per-agent `scope` ("project" | "global") instead of a single install mode
- Global installation detection in `init` command: prompts to edit global, create project, or cancel when global config exists
- Wizard store tracks per-agent scope selections with `setAgentScope()` action
- Step-agents supports scope toggle per agent
- Confirm step displays per-agent scope summary (e.g., "2 project, 1 global")
- Config-types writer emits `AgentScopeConfig` type in generated config-types.ts
- Config writer serializes per-agent scope to config.ts
- Source switcher handles scope migration during edit sessions
- `buildAgentConfigs()` test helper for generating AgentScopeConfig arrays
- D-37 design doc (`todo/D-37-dual-installation.md`)
- Refactor items R-05 (centralize wizard hotkeys) and R-06 (slim down ResolvedSkill)

## Changed

- `ProjectConfig.agents` type changed from `AgentName[]` to `AgentScopeConfig[]` (breaking internal type change)
- Zod schema `projectConfigLoaderSchema` validates agents as objects with `name` and `scope` fields
- Config generator builds `AgentScopeConfig[]` from wizard selections
- Config merger handles scope-aware agent merging (project + global deduplication)
- Local installer propagates per-agent scope through installation pipeline
- Resolver and stack compiler consume `AgentScopeConfig[]`
- Agent recompiler extracts agent names from scope config objects
- Uninstall command reads agent names from `AgentScopeConfig[]`
- All unit, integration, and E2E tests updated for new agents format
- E2E `createEditableProject` converts `AgentName[]` to `AgentScopeConfig[]` internally
- CLAUDE.md test factory list expanded with 20+ new mock helpers

## Fixed

- Compile E2E global fallback test now passes (unmarked from `.fails`)
- Init `--refresh` flag test no longer hangs (added `--source` to prevent GlobalConfigPrompt)
- Doctor E2E inline configs updated for AgentScopeConfig format
- Uninstall `collectConfiguredAgents` correctly reads `.name` from objects

## Completed

- D-37: Install mode UX redesign (dual-scope agent installation)
- D-65: Init/edit scope with global config detection and prompt
- B-09: Custom skill retains `source: "local"` through edit flow
