# Release 0.74.0 (2026-03-13)

## Changed

- Replace `writeProjectSourceConfig()` with `generateConfigSource()` in all callers — config writes now produce proper TypeScript with `import type { ProjectConfig }` and `satisfies ProjectConfig`
- `saveSourceToProjectConfig()` takes explicit `name` parameter instead of silently defaulting to `path.basename(projectDir)`
- `addSource()` and `removeSource()` require an existing config with `name` — throw if no project config found

## Removed

- `writeProjectSourceConfig()` function from `config.ts`
- `ProjectSourceConfig` type alias from `config.ts` (was already `Partial<ProjectConfig>` with `@deprecated` tag)
- Both symbols removed from `configuration/index.ts` exports
