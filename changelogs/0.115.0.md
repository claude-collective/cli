# Release 0.115.0 (2026-04-09)

## Fixed

- **Excluded skills no longer shown as selected in edit mode**: Skills with `excluded: true` in config were incorrectly included in `currentSkillIds`, making them appear as selected in the build step and silently losing the excluded tombstone on no-op edits.
- **Dual-entry skills correctly shown as selected**: When a skill has both an excluded global tombstone and an active project entry (from Gâ†’P scope toggle), the active entry is no longer filtered out. The exclusion filter now only applies to plugin-discovered skills, not config entries.
- **Scope toggle deselect no longer removes from global config**: `detectConfigChanges` now diffs against `oldConfig.skills` instead of the broader `currentSkillIds` set, preventing globally-discovered skills from appearing as false removals.
- **Agent scope map no longer poisoned by excluded tombstones**: The `agentScopeMap` in `writeConfigAndCompile` now filters excluded entries, preventing tombstones from overwriting active scope assignments via Map last-entry-wins semantics.
- **Gâ†’P scope toggle preserves global installation (override model)**: Toggling agents or ejected skills from global to project scope now adds to project without deleting from global. The global installation remains untouched for other projects.

- E2E tests for excluded skill behavior (`edit-wizard-excluded-skills.e2e.test.ts`, 5 tests)
- E2E tests for scope deselect integrity (`scope-change-deselect-integrity.e2e.test.ts`, 3 tests)
