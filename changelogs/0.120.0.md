# Release 0.120.0 (2026-04-13)

## Added

- Dual-scope badges in build step: when a skill exists at both global and project scope (e.g. after G→P toggle), both scope badges are shown side by side (D-191)
- Global agent toggle guard: toggling a globally-installed agent from project scope now shows a toast instead of silently modifying global config (D-196)
- E2E test for global agent toggle guard in edit wizard

## Fixed

- Project skill misreported as global after external install: `buildSkillConfigForId` now prefers project-scoped entries over global when duplicates exist (D-198)
- Init mode without pre-existing global config no longer creates unnecessary excluded tombstones for agents

## Changed

- Config-writer tests verify skills and agents appear in both `// global` and `// project` sections when present at both scopes
- User-journey tests cover the full init→split→inline pipeline for dual-scope skills
