# Release 0.94.0 (2026-03-28)

## Changed

- **Agent template restructuring (D-138):** All 20 agent definitions renamed to a consistent `identity.md` / `playbook.md` / `output.md` / `critical-requirements.md` / `critical-reminders.md` structure. Previous `intro.md` → `identity.md`, `workflow.md` → `playbook.md`, `output-format.md` → `output.md`, `examples.md` merged into `output.md`. Content expanded across critical-requirements and critical-reminders for all agents.
- Agent liquid template, compiler, stack-plugin-compiler, consts, and types updated to match new file naming convention.
- `new:agent` command scaffolds agents using updated template structure.

## Fixed

- **D-161:** Exhaustive (radio) categories in edit mode could deselect locked global skills via radio swap. The exclusive toggle now rejects the swap when it would evict a locked skill.
- **D-160:** Sources step only showed the installed check (✓) for project-scope plugin skills. Global-scope plugins are now discovered from `~/.claude/settings.json` in addition to the project directory, matching the existing global+project merge pattern for local skills.
- **E2E: node-pty undefined coercion:** `TerminalSession` now strips `undefined` values from the env object before passing to `pty.spawn`. Node-pty was converting `undefined` to the string `"undefined"`, causing `CC_SOURCE="undefined"` to leak into child processes and bypass the `BUILT_IN_MATRIX` code path.

## Added

- E2E tests for default source / `BUILT_IN_MATRIX` code path (D-123/D-124).
- Dual-scope edit lifecycle E2E tests (display, locking, agent integrity, config split).
- Agent findings: `init-missing-global-compile.md`, `matrix-loading-performance.md`.
