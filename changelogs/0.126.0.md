# Release 0.126.0 (2026-04-14)

## Fixed

- **Global plugin preserved on scope migration** — When migrating a globally-installed plugin skill to project-scope eject, the mode-migrator no longer uninstalls the global plugin registration. The global plugin remains available for other projects; the project config tombstone (`excluded: true`) hides it from the current project.
- **Source step navigation order** — `buildSourceRows()` now stable-sorts rows (global readOnly first, global editable, project last) to match the visual grouping from `groupRowsByScope()`. Previously keyboard navigation jumped between visual sections because array indices didn't match render order.
- **Local-installer test isolation** — Tests no longer write to or clean up the real `~/.claude-src/` directory. Uses a fake `HOME` env var instead.

## Changed

- **Removed lock icon from build step** — The `🔒` icon on globally-installed skills was redundant with the `[G]` scope badge and misleading after scope toggles (the project copy is editable but the icon suggested otherwise). The toast guard already prevents invalid actions.
- **Removed eject indicator from build step** — The `⏏` icon on ejected skills was visual noise in the build step; source mode is visible on the sources step where it matters.
- **Unified scope badge colors** — Both `[G]` and `[P]` scope badges now use the same warning color for visual consistency.
