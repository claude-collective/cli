# Release 0.73.0 (2026-03-12)

## Changed

- Extract `matrix-provider` module from Zustand matrix-store — pure module-level state replaces reactive store
- Extract `skill-resolution` module from matrix-loader — `mergeMatrixWithSkills`, `buildResolvedSkill`, and relationship resolution now in dedicated file
- Generate built-in matrix at build time from skills source (`types/generated/matrix.ts`)
- Remove extensible schema infrastructure (`customExtensions`, `extensible*Schema`, `extendSchemasWithCustomValues`, `resetSchemaExtensions`) — generated enums replace runtime extension
- Remove `isValidSkillId` — skill ID validation now uses generated `z.enum(SKILL_IDS)` directly
- `metadataValidationSchema` now uses `z.enum(CATEGORIES)` and `z.enum(SKILL_SLUGS)` for strict IDE validation
- Simplify codegen scripts: replace custom serialization with `JSON.stringify`, add `sortedGroupBy` helper, run prettier as post-step
- Refactor `generateStackAgentConfig` from imperative loop to declarative `.map()` + `.join()` with extracted `formatSkillUnion` helper
- Simplify `info.ts` skill lookup — extract slug resolution into intermediate variable

## Added

- `custom-metadata.schema.json` — relaxed schema for custom skill metadata (plain string category/slug, no `additionalProperties: false`)
- `utils/type-guards.ts` — `isCategory()`, `isDomain()`, `isAgentName()` type guards replacing boundary casts
- Unit tests for `generate-source-types.ts` functions (`sortedGroupBy`, `resolveStack`, `extractSkills`, `extractAgents`, `generatePhase1`, `generatePhase2`)

## Removed

- `matrix-store.ts` — Zustand store replaced by `matrix-provider.ts` module
- `SetupPair` type, `requiresSetup`, `providesSetupFor` fields, `resolveSetupPairs()`, `validateSetupUsage()` — unused setup infrastructure with no configured data
- `DOMAIN_VALUES` and `CATEGORY_VALUES` re-exports from schemas.ts
- `extensibleDomainSchema`, `extensibleSkillIdSchema`, `extensibleCategorySchema`, `extensibleAgentNameSchema`
- `validateExtensibleRecordKeys` superRefine helper and all category/domain key validation on record schemas

## Test Improvements

- Centralize inline mock data to `mock-data/` folder (mock-skills.ts, mock-matrices.ts, mock-categories.ts)
- Replace fake skill IDs (`web-skill-a`, `web-skill-b`) with valid `SkillId` union members in matrix-resolver tests
- Move multi-source integration test data (skills, categories, matrix builder) to shared mock files
