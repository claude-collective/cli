# Release 0.111.0 (2026-04-06)

## Added

- Global-first setup guide (`docs/guides/global-first-setup.md`) explaining the recommended workflow: global scope as default, project scope for overrides
- E2E test verifying eject skills are copied to project dir when scope is toggled during init
- New TODOs: D-171 through D-178 covering edit output styling, scope-pure config types, sources step UX, uninstall bug, and stacks as project hero

## Fixed

- **Project config no longer includes global agent stack entries** — global agents' skill mappings stay in the global config where they belong. Previously, the inlined project config included global stack entries that couldn't be changed from the project context
- **Dual-pass scope-filtered compilation** for init and edit commands — global agents compile from the global config, project agents from the project config. Matches the pattern already used by the `compile` command. Prevents the project pass from overwriting correctly compiled global agents with empty skill lists
- **Eject skills now copy across scopes during init** — when a globally-ejected skill is toggled to project scope, the files are copied from `~/.claude/skills/` to `<project>/.claude/skills/`. Previously the local-skill check short-circuited without copying
- **Terminal clearing after wizard exit** — uses Ink's `clear()` method plus scrollback buffer clear (`\x1b[3J`) for reliable screen clearing
- Flaky category-grid left arrow test stabilized with increased render delay

## Changed

- Simplified SkillAgentSummary component — removed unused scope-change tracking sets
- Removed obsolete tracker files (excluded-skills-tracker.md, new-skills-tracker.md)
