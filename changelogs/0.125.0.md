# Release 0.125.0 (2026-04-13)

## Changed

- **D-168: E2E test audit** — Full sweep of 86 E2E test files against documented standards. 20 audit agents identified anti-patterns, 7 fix agents resolved them across ~45 test files. All 519 tests pass.

### Constants & Hardcoded Strings

- Added 9 new `STEP_TEXT` constants (plugin UI text, wizard prompts, toast messages)
- Replaced hardcoded UI strings with constants in 19 test files
- Replaced `".claude-plugin"` with `SOURCE_PATHS.PLUGIN_MANIFEST_DIR` in 6 files
- Replaced `"agents"` with `DIRS.AGENTS` in 2 lifecycle files
- Removed unused imports in 4 interactive test files

### Matcher Adoption

- Replaced raw `fileExists`/`readTestFile`/`directoryExists` with content-aware matchers (`toHaveConfig`, `toHaveCompiledAgent`, `toHaveCompiledAgentContent`, `toHaveEjectedTemplate`, `toHaveSettings`, `toHaveLocalSkills`, `expectCleanUninstall`) in 12 files
- Fixed conditional assertions (`if (await fileExists(x)) { assertions }`) that silently skip in `plugin-uninstall-edge-cases`
- Added missing `../matchers/setup.js` imports in 3 files

### Fixture Extraction

- Replaced inline `mkdir` + `createPermissionsFile` with `createTestEnvironment()` in 5 files (fixes missing `createPermissionsFile` in `init-global-preselection-confirm` — potential flaky hang)
- Fixed temp dir leaks from `ProjectBuilder.editable()` in 3 files
- Replaced inline metadata YAML with `createLocalSkill()`/`addForkedFromMetadata()` in 2 files
- Replaced inline agent stub `writeFile` with local helpers in 6 files

### Shared Helpers

- Extracted `expectNoDuplicates()` to `e2e/assertions/config-assertions.ts` (was duplicated in 2 lifecycle tests)
- Added `EditWizard.completeFromBuild()` method (was duplicated in 3 interactive tests)
- Added `ProjectBuilder.dualScopeWithImport()` for dual-scope compile verification
- Rewrote `unified-config-view.e2e.test.ts` from 139 lines to 51 lines using new builder

### Missing Assertions

- Added `{ timeout: TIMEOUTS.PLUGIN_TEST }` to all 9 `it()` blocks in `init-wizard-plugin` (default 30s insufficient for plugin operations)
- Added negative assertions after skill removal in 3 interactive tests
- Added negative error assertions after edit phase in `init-then-edit-merge`
- Fixed comment/code mismatch in `edit-wizard-plugin-operations` add-skill assertion

### Standards

- Added 6 new rules to `.ai-docs/standards/e2e/anti-patterns.md`: conditional assertion guard, inline mkdir for dual-scope, ProjectBuilder temp dir cleanup, inline agent stubs, shared helper extraction threshold, ProjectBuilder extension pattern

### Documentation

- 3 validation passes on reference docs: accuracy fixes, line number stripping, cross-reference verification
- Updated DOCUMENTATION_MAP with validation history
