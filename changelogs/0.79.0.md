# Release 0.79.0 (2026-03-21)

## Added

- `claudePluginMarketplaceUpdate()` utility in `exec.ts` for refreshing stale marketplace clones
- Mixed install mode in `init.tsx` — when skills have mixed sources, local skills are scope-split to project/global directories while plugin skills install via Claude CLI
- Marketplace auto-update on `cc init` when marketplace clone already exists
- New E2E test suites: edit wizard detection, default source path, scope split verification, global scope lifecycle, init-then-edit merge, scope-aware local copy
- Scope awareness rules added to CLAUDE.md (NEVER/ALWAYS sections)

## Changed

- `source-loader.ts` now always merges both global and project local skills instead of falling back to global only when project has none
- `wizard-store.ts` preserves saved source from config over computed `primarySource` (`saved?.source ?? primarySource` instead of `primarySource ?? saved?.source`)
- `config-generator.ts` filters global agents' stacks to only reference global-scoped skills; domains route exclusively to global config
- `config-merger.ts` lets new source win when provided (sourceFlag takes precedence); agents merge by name preserving scope changes
- `config-types-writer.ts` includes `config.domains` in domain type generation
- `installation.ts` simplified — removed redundant local-mode branches from `resolveInstallPaths()`
- `local-installer.ts` uses `os.homedir()` as baseDir for global skill deletion
- `mode-migrator.ts` splits migrations by scope (project vs global) before copy/delete operations
- `edit.tsx` merges discovered plugin skills with config skills; deletes old agent files after scope change; scope-aware skill deletion
- `uninstall.tsx` derives plugin scope from per-skill config instead of directory heuristic; simplified `shouldRemoveSkill` to check only `forkedFrom` presence
- E2E tests updated for scope-aware assertions (source switching, dual-scope integrity, cross-scope lifecycle)

## Fixed

- Global-scoped local skills no longer ignored when project also has local skills
- Compile dual-pass no longer overwrites global agents with zero-skill versions during project pass
- Global plugin skills now discovered separately and merged during compilation
- `cc diff`, `cc doctor`, `cc outdated`, `cc update` now check both project and global local skill directories
- Config source no longer silently overwritten by existing config on re-init
- Domain types now include user-selected domains from config, not just skill-derived domains
