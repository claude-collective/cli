# Release 0.58.0 (2026-03-06)

## Added

- Per-skill scope toggle: S key in build step switches focused skill between project and global scope
- Scope badge [P]/[G] shown on focused skill tag in build step
- Confirm step displays per-skill scope summary (e.g., "2 project, 1 global") instead of single install mode
- `createMinimalInstallation()` E2E test helper for temp directory setup
- `createMockSkillConfig()` test factory for SkillConfig objects
- D-73 deferred task: split init-wizard E2E tests for parallel execution
- Schema removal audit research doc (D-71)

## Changed

- Help modal: "Global Toggles" section renamed to "Toggles"
- `new skill` command: skip `detectInstallation()` when `--output` flag is provided, fixing 19 E2E test failures in `new marketplace` and `new skill` flows
- E2E `WIZARD_LOAD_TIMEOUT_MS` reduced from 20s to 10s
- Test suite updated for `SkillConfig[]` format (replaces `installMode` + `sourceSelections`)
- Install mode integration tests rewritten for per-skill derived mode
- Category grid: simplified ref indirection for `focusedSkillId` callback

## Fixed

- `new marketplace` failing in temp dirs due to `detectInstallation()` check when delegating to `new skill --output`
- 28 E2E test failures: confirm step text casing, help modal section titles, removed `--global` flag tests
- Category grid `focusedSkillId` never being set due to stale ref indirection

## Removed

- `--global` flag tests from init wizard E2E (flag was removed in 0.57.0)
