# Release 0.135.0 (2026-04-17)

E2E test infrastructure refactor: fixtures, page objects, and matchers overhauled for reliability and ergonomics. All 40 existing E2E tests updated to consume the new APIs. No production code changes.

## Changed

- **`e2e/fixtures/dual-scope-helpers.ts` rewritten**. `initProject` and `initProjectAllGlobal` (Phase B of `setupDualScopeWithEject`) now drive the dashboard → Edit flow when a prior global install exists, instead of re-running a fresh init wizard that would reject the pre-existing state. Behavior-preserving refactor — same end state, realistic user path.

- **Page objects hardened with stability waits**:
  - `e2e/pages/base-step.ts` — new `waitForStableRender()` called before key-press methods to prevent race conditions where React effects hadn't fired yet. Also `pressEnterWaitNewFrame()` helper.
  - `e2e/pages/wizards/edit-wizard.ts` — three-sentinel `waitFor` chain on launch so tests only see the edit wizard once the store has hydrated and the first real frame has rendered.
  - `e2e/pages/steps/build-step.ts` — `toggleScopeOnFocusedSkill`, `toggleFocusedSkill`, navigation helpers all await stable render before pressing keys. Eliminates intermittent flake where handlers silently no-op'd on mid-animation frames.
  - `e2e/pages/dashboard-session.ts` — wait-before-select hardening; `selectEdit()` / `selectCompile()` / etc. all await menu fully rendered before Enter.

- **`e2e/pages/constants.ts` extended** with additional `DIRS`, `FILES`, `STEP_TEXT`, `EXIT_CODES`, `TIMEOUTS` entries used by new tests (under 0.136.0) and by fixture migrations.

- **`e2e/pages/terminal-screen.ts`** — added `readStyledLines()` and other screen-buffer utilities for style-aware assertions.

- **`e2e/matchers/agent-matchers.ts`** extended with new assertion helpers (`toHaveSkillCopied`, `toHaveCompiledAgent`, `toHaveCompiledAgentContent`). `setup.ts` registers the new matchers.

- **`e2e/fixtures/project-builder.ts`** — small ergonomic additions for source/fixture composition.

- **40 existing E2E test files updated** to consume the new fixture/page/matcher APIs:
  - `e2e/commands/*` — 6 files (compile, dual-scope, eject, relationships, uninstall, validate)
  - `e2e/interactive/*` — 15 files (edit-wizard + init-wizard variants)
  - `e2e/lifecycle/*` — 16 files (dual-scope edits, exclusion, scope-change, cross-scope, etc.)
  - `e2e/integration/custom-agents.e2e.test.ts`
  - `e2e/smoke/pom-framework.e2e.test.ts`

  Updates are mechanical: swap inline fixture construction for helpers, adopt new page object APIs, use canonical constants from `e2e/pages/constants.ts`.

## Notes

- No new tests added in this release. New lifecycle test coverage (7 files) ships in 0.136.0 alongside documentation updates.
- No production code changes. Commit log in this release is entirely test-layer.
