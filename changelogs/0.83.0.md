# Release 0.83.0 (2026-03-25)

## Added

- **Inline global config in project config (D-129)** — project `config.ts` is now a self-contained readable snapshot. Global skills and agents are inlined with `// global` and `// project` comment separators instead of hidden behind `...globalConfig.skills` / `...globalConfig.agents` spreads. No more `import globalConfig` in the project config.
- **Standalone project config-types** — project `config-types.ts` is generated as standalone (all types defined locally) instead of importing from the global config-types.
- **Deduplicated selectedAgents and domains** — merged arrays from global + project configs are now deduplicated.

## Fixed

- **Multi-skill category stack generation** — `Object.fromEntries` with category as key was overwriting duplicate categories, so multi-select categories (e.g., `web-styling` with CVA + Tailwind + SCSS) lost all but the last skill. Skills are now grouped by category before building stack entries.
- **SkillAssignment[] arrays in generated config** — stack values in both global and project configs are now proper `SkillAssignment[]` arrays instead of bare strings that violated the type. The generated `config-types.ts` emits `SkillAssignment[]` to match.
- **Project stack excludes global agents** — the project config's stack now only includes project-scoped agents. Global agents' stack entries belong in the global config only.
- **Scalar field deduplication** — when both global and project configs share scalar fields (e.g., `source`, `marketplace`), project values take precedence and each key appears exactly once.

## Removed

- **`compactStackForYaml`** — dead function from the YAML config era, replaced by `compactStackAssignments` which preserves `SkillAssignment[]` arrays.
- **`writeProjectConfigTypes`** — dead function in local-installer, replaced by `writeStandaloneConfigTypes`.
