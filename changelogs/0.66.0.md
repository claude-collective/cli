# Release 0.66.0 (2026-03-11)

## Changed

- **Relationship rules now use slugs instead of canonical IDs (R-04 Phase 2)** — all 7 relationship types (`ConflictRule`, `DiscourageRule`, `Recommendation`, `RequireRule`, `AlternativeGroup`, `CompatibilityGroup`, `SetupPair`) reference skills by human-readable slugs (`"react"`, `"zustand"`) instead of canonical IDs (`"web-framework-react"`, `"web-state-zustand"`). The resolver maps slugs to canonical IDs at load time via `slugMap.slugToId`.
- `default-rules.ts` rewritten from canonical IDs to slugs — ~100 skill references updated
- `ResolveId` type and `resolveToCanonicalId()` in `matrix-loader.ts` now accept `SkillSlug` input
- Recommends lookup in `buildResolvedSkill` compares `r.skill === skill.slug` instead of `skill.id`
- Zod schema variable renamed from `skillRefInYaml` to `skillRefInRules` — uses `skillSlugSchema` for validation
- `slug` field now required in `metadata.schema.json` (R-04 Step 7 — was deferred, now finalized)

## Added

- E2E test file `e2e/commands/relationships.e2e.test.ts` — 9 tests covering conflict detection, require enforcement, recommend marking, slug resolution, discourages, and combined rules
- `createE2ESource()` now accepts optional `relationships` parameter for custom skill rules in E2E tests
- Implementation plan at `todo/R-04-phase-2-slug-migration.md`

## Fixed

- R-04 TODO tracker updated: Phase 2 and Step 7 marked complete, R-04 overall status set to Done
