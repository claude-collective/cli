# Release 0.105.0 (2026-04-05)

## Added

- Excluded skill/agent concept — deselecting a globally-installed skill or agent during project edit marks it `excluded: true` in the project config rather than removing it, preserving the entry for re-selection later
- Scope toggle creates excluded entries — moving a previously-installed global skill/agent to project scope adds both a project entry and an `excluded: true` global entry, so the compiler knows to skip the global version for this project
- Scope-aware confirm step — the "Ready to install" summary now detects new vs existing items by `id:scope` pair, so moving React from global to project correctly shows it as new in Project and existing in Global
- Inherited globals in confirm step — globally-installed skills/agents that were moved to project scope still appear in the Global section as existing
- Inherited globals in sources step — globally-installed skills from `skillConfigs` now appear in the sources step alongside newly selected skills
- `filterExcludedEntries` exported from agent-recompiler for reuse in doctor and other commands

## Changed

- `applySkillRemoval` and `applyAgentToggle` now check `installedSkillConfigs`/`installedAgentConfigs` before marking entries as excluded — on fresh init (no prior installation), deselection removes entries entirely
- `toggleSkillScope`/`toggleAgentScope` only create excluded global entries when the skill/agent was previously installed globally
- Doctor command uses `filterExcludedEntries` for `checkAgentsCompiled` and `checkSkillsInstalled`, and filters excluded skill IDs from the stack in `checkSkillsResolved`
- Inlined project config now includes the full global stack (merged global + project agents' stack entries), making it self-contained
- `writeScopedConfigs` passes the effective global config (with preserved stack) to the project config writer instead of the split partition
- Excluded globals render under `// global` comment in generated config files instead of under `// project`

## Fixed

- Fresh init no longer marks deselected agents/skills as `excluded: true` — they are simply removed since nothing was previously installed
- Confirm step now shows `+` prefix (green, new) for all items on fresh init instead of bullet (existing)
- Doctor no longer reports false-positive "not found" for excluded skills referenced in the stack
- Inlined project config no longer loses the global agents' stack, fixing "No skills configured" in doctor after dual-scope init
- `generateProjectConfigFromSkills` no longer picks up excluded agent config when looking up active agents by name

## Removed

- Dead code: `getAllDomainsFromCategories`, `SkillLookupEntry` type, `FRAMEWORK_CATEGORY_ID` constant, unused `categories` parameter in `useCategoryGridInput`, unused `useEffect` import in step-agents, commented-out Toast component

## Test Improvements

- `init-flow.integration.test.ts` now mocks `os.homedir()` at module level — prevents deleting the user's real `~/.claude-src/` directory during test runs
- `config.test.ts` isolates HOME in `resolveSource` tests to avoid leaking through to real global config
- E2E assertion updates for excluded concept, property reordering normalization, and skills-array-only accumulation checks
