# Release 0.88.0 (2026-03-26)

## Changed

- **`synthesizeCategory()`** now logs a verbose warning when auto-generating placeholder categories for undefined entries.

## Fixed

- **Plugin category separation** (D-149): `category` was removed from `PluginManifest` — plugin manifests (`plugin.json`) describe plugin structure, not skill metadata. Categories belong in `metadata.yaml` and flow to `MarketplacePlugin` during marketplace generation. Removed `inferCategoryFromPluginName` and `deriveCategoryFromName` which inferred categories from plugin names instead of reading from metadata.

## Removed

- **`inferCategoryFromPluginName()`** from `marketplace-generator.ts` — categories come from skill metadata, not name inference.
- **`deriveCategoryFromName()`** from `plugin-manifest.ts` — same principle.
- **`category` field from `PluginManifest` type** and all plugin manifest Zod schemas — plugin.json doesn't carry skill metadata.
- **`category` parameter from `generate*PluginManifest` functions** — manifests no longer include category.
- **Unnecessary test mocks** (D-148): Removed `logger` and `errors` mocks from `plugin-discovery.test.ts` and `plugin-finder.test.ts` that were never asserted against.
