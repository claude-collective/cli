# Release 0.88.0 (2026-03-26)

## Changed

- **`PluginManifest.category` is now required and typed as `Category`** (D-149): No more inferring categories from plugin names. The `category` field on `PluginManifest` and `MarketplacePlugin` is a required field validated against the generated `Category` union at all parse boundaries (Zod `z.enum(CATEGORIES)`). Removed `inferCategoryFromPluginName` and `deriveCategoryFromName`.
- **`generateSkillPluginManifest`, `generateAgentPluginManifest`, `generateStackPluginManifest`** now require an explicit `category` parameter instead of deriving it from the plugin name.
- **`synthesizeCategory()`** now logs a verbose warning when auto-generating placeholder categories for undefined entries.

## Fixed

- **Plugin category type safety**: Categories were passed as `string` throughout the plugin system, allowing invalid values to flow silently. Now validated against the `Category` union at all three Zod schema parse sites (`pluginManifestSchema`, `pluginManifestValidationSchema`, `plugin-validator.ts`).

## Removed

- **`inferCategoryFromPluginName()`** from `marketplace-generator.ts` — categories must come from metadata, not name inference.
- **`deriveCategoryFromName()`** from `plugin-manifest.ts` — same principle.
- **Unnecessary test mocks** (D-148): Removed `logger` and `errors` mocks from `plugin-discovery.test.ts` and `plugin-finder.test.ts` that were never asserted against.
