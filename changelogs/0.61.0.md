# Release 0.61.0 (2026-03-08)

## Added

- Dual-pass compile: `compile` now detects both global (`~/.claude-src/`) and project (`./.claude-src/`) installations and runs independent compilation passes for each
- `writeProjectConfig()` E2E helper — typed factory for writing `config.ts` files to disk, defaults `skills` and `agents` to `[]`
- `createDualScopeProject()` E2E helper — creates paired global + project installations for dual-scope testing
- Dual-scope E2E tests: 6 tests covering both-locations compile, skill scope isolation, single-installation fallbacks, and verbose output
- Matrix store design doc (`docs/features/proposed/matrix-store-design.md`)

## Changed

- Compile command uses `detectGlobalInstallation()` + `detectProjectInstallation()` instead of single `detectInstallation()`
- Zero-skills is non-fatal per pass — only errors when ALL passes find zero skills
- Verbose output labels each pass (`"Compiling global agents..."` / `"Compiling project agents..."`)
- All E2E config writes migrated from inline `writeFile` + `JSON.stringify` to `writeProjectConfig()` helper
- `agents` format in E2E tests normalized to `AgentScopeConfig[]` (was bare string arrays in some tests)
- CLAUDE.md updated: `writeProjectConfig` added to config factories list and commit checklist

## Removed

- `runPluginModeCompile` — replaced by unified `runCompilePass`
- `loadProjectConfigFromDir` import in compile command — config loading is handled per-pass by `discoverAllSkills`
- `getStackSkillIds` import in compile command — no longer needed
