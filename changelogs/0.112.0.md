# Release 0.112.0 (2026-04-08)

## Fixed

- **Agent toggle-off preserves selectedAgents for globally-installed agents** — toggling off a global agent now keeps it in `selectedAgents` with an `excluded` tombstone in `agentConfigs`, so `SelectedAgentName` stays correct for other projects sharing the global config
- **Sources step marks globally-installed skills as readOnly** — re-scoped skills show a locked global row + editable project row; excluded global skills show a single locked row. Prevents accidental source changes to global installations from project scope
- **Confirm step suppresses global removal markers during init** — deselecting a global preselection during project init no longer shows misleading removal diff markers
- **ProjectAgentName narrows to project-scoped agents** — generated type union excludes global agents, so `stack` keys only accept agents available in the current scope
- **Project config stack excludes global agent entries** — global agents' skill mappings stay in the global config only
- **Edit command uses scope-aware plugin changes** — uninstall respects per-skill scope during edit operations

## Added

- E2E test: confirm step doesn't show deselected global skills as removed during project init
- E2E test: excluded global agents included in SelectedAgentName union
- Agent findings for E2E retry isolation and re-scoped skill duplicate row rendering
