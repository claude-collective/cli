# Release 0.52.0 (2026-02-28)

## Added

- **Generated `config-types.ts` for editor type safety** — New `ts-config-types-writer` module generates a zero-dependency `config-types.ts` file alongside `config.ts` during `init` and `edit`. Contains typed string unions (`SkillId`, `AgentName`, `Domain`, `Subcategory`) and a `ProjectConfig` interface derived from the resolved marketplace matrix, giving users full editor autocomplete and type checking with just `import type`.
- **`STANDARD_FILES.CONFIG_TYPES_TS` constant** — Standard filename constant for the generated types file.

## Changed

- **Config output uses `satisfies ProjectConfig` instead of `defineConfig()`** — Generated `config.ts` now exports a plain object literal with `satisfies ProjectConfig` and `import type { ProjectConfig } from "./config-types"`, eliminating the runtime dependency on `@agents-inc/cli/config`. This fixes the root cause of B-08 (uninstall failing because jiti couldn't resolve the `defineConfig` import in production).
- **Test helper `readTestTsConfig` rewritten to use jiti** — Replaced the fragile regex-based `parseTsConfigContent` with a proper jiti-based loader that handles `defineConfig()`, `satisfies`, and plain exports. The old `parseTsConfigContent` export has been removed.
- **D-46 task plan condensed** — Replaced the 860-line phased implementation plan with a focused summary of completed work and remaining tasks (custom entity discovery in generated types).

## Fixed

- **B-08 root cause identified** — Uninstall not removing compiled agents was caused by `loadProjectSourceConfig` returning null because jiti couldn't resolve the `@agents-inc/cli/config` import from `defineConfig()`. The switch to `satisfies ProjectConfig` eliminates this import entirely.
