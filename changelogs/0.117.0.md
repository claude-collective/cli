# Release 0.117.0 (2026-04-09)

## Changed

- **Test assertion quality sweep across 133 test files (5078 tests)**
  - Replaced 305 `toBeDefined()` with `toStrictEqual` / specific value assertions
  - Replaced 20 `toBeTruthy()` with exact type/value checks
  - Replaced 79 `toBeGreaterThan(0)` with exact `toHaveLength(N)` counts
  - Converted 19 positive `toHaveBeenCalled()` to `toHaveBeenCalledWith(exactArgs)`
  - Eliminated all bare `toHaveConfig()` and `toHaveCompiledAgents()` E2E calls
  - Added `skillIds`, `agents`, and `source` parameters to all `toHaveConfig` assertions
  - Replaced `toHaveCompiledAgents()` with specific `toHaveCompiledAgent("name")` + content checks
  - Replaced 189 `toContain` loops with `toStrictEqual` on sorted arrays
  - Replaced 76 `toHaveLength` count-only checks with exact content verification
  - Replaced 41 hardcoded strings with `STEP_TEXT`, `DIRS`, `FILES`, `EXIT_CODES` constants
  - Replaced 79 raw `readTestFile`/`fileExists` in E2E test bodies with custom matchers
  - Added 60 negative assertions (verify removed items are gone after uninstall/deselect)
  - Added 62 `toHaveCompiledAgentContent` assertions verifying agent file content
  - Replaced 47 bare exit code numbers with `EXIT_CODES.*` constants
  - Fixed 9 missing `matchers/setup.js` imports in E2E tests
  - Removed 5 duplicate local helper functions (replaced with shared imports)
  - Added 11 new tests for previously untested paths

## Removed

- 12 misclassified unit tests from `install-mode.integration.test.ts` (duplicated in dedicated unit test files)
- 3 dead code exports from `helpers.ts` (`createMockMultiSourceSkill`, `createMockSkillSource`, `createTestSkill`)

## Fixed

- `compile-flow.test.ts`: Fixed silent pass-through on compile errors (tests were passing with 0 assertions executed)
- `global-scope-lifecycle.e2e.test.ts`: Fixed agent compilation assertions checking wrong directory (projectDir vs fakeHome)
- `dual-scope-edit-mixed-sources.e2e.test.ts`: Fixed overly broad `not.toContain('"source":"eject"')` catching excluded global entries
- `dual-scope-edit-source-changes.e2e.test.ts`: Same excluded global entry fix
- `init-end-to-end.integration.test.ts`: Fixed wrong agent scope (project vs global), skill ordering, stack compaction shape
- `wizard-store.test.ts`: Fixed `excluded: undefined` vs missing property distinction in `toStrictEqual`
- `local-installer.test.ts`: Fixed mock data producing empty agents when test expected populated agents
