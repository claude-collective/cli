# Release 0.86.0 (2026-03-26)

## Added

- **Operations layer** (D-145): 26 composable, typed operations in `src/cli/lib/operations/` that centralize repeated command sequences. Each operation wraps lower-level lib functions into a single call with structured result types.
  - Core operations: `loadSource`, `detectProject`, `loadAgentDefs`, `compileAgents`, `copyLocalSkills`, `installPluginSkills`, `uninstallPluginSkills`, `ensureMarketplace`, `writeProjectConfig`
  - Skill operations: `compareSkillsWithSource`, `collectScopedSkillDirs`, `findSkillMatch`, `updateLocalSkills`, `generateSkillDiff`, `discoverInstalledSkills`, `resolveSkillInfo`
  - Lifecycle operations: `executeInstallation`, `recompileProject`, `ejectProject`, `uninstallProject`, `searchSkills`, `importSkill`
  - Scaffold operations: `scaffoldSkill`, `scaffoldAgent`
  - Config operations: `detectBothInstallations`, `detectConfigChanges`, `getDashboardData`, `migratePluginSkillScopes`
- **Shared `truncateText()` utility** in `src/cli/utils/string.ts` — replaces 3 duplicate implementations
- **`warn()` `suppressInTest` option** — suppresses expected matrix warnings during test runs without silencing genuine warnings
- **`buildSourceSkillsMap()`** — shared helper extracted from duplicated source skills map building in compare-skills and diff

## Changed

- **All 15 command files refactored** to use operations layer instead of direct lib imports:
  - `compile.ts`: ~40% line reduction (349 -> 210 lines), skill discovery extracted to `discoverInstalledSkills`
  - `edit.tsx`: change detection extracted to `detectConfigChanges`, plugin scope migration to `migratePluginSkillScopes`
  - `init.tsx`: dashboard data extraction to `getDashboardData`
  - `update.tsx`: skill update loop extracted to `updateLocalSkills`, matching to `findSkillMatch`
  - `diff.ts`: diff logic extracted to `generateSkillDiff`, dir collection to `collectScopedSkillDirs`
  - `doctor.ts`: uses `detectProject` operation, consolidated double config loading
  - `eject.ts`: eject logic extracted to `ejectProject` operations
  - `uninstall.tsx`: full extraction to `uninstallProject` operations (417 -> 256 lines)
  - `search.tsx`: fetch/filter/copy extracted to `searchSkills` operations
  - `info.ts`: skill resolution extracted to `resolveSkillInfo`
  - `import/skill.ts`: GitHub parsing + import logic extracted (424 -> 225 lines)
  - `new/skill.ts`: scaffold generation extracted (382 -> 197 lines)
  - `new/agent.tsx`: meta-agent loading + invocation extracted (296 -> 187 lines)
  - `outdated.ts`: already clean, minimal changes
  - `new/marketplace.ts`: updated imports for scaffold-skill operations
- **`compare-skills.ts`** now uses `collectScopedSkillDirs()` internally instead of duplicating path resolution
- **`compile-agents.ts`** enhanced to auto-build `agentScopeMap` when `scopeFilter` is set

## Fixed

- **E2E test performance**: Added `retry: 0` to 8 `it.fails()` tests that were wasting ~2-4 minutes each on pointless retries (known ENOENT bug D-152)
- **Test noise**: Matrix "skill not found" warnings suppressed during test runs via `warn({ suppressInTest: true })`
- **Unused type imports** cleaned up in `doctor.ts` (`AgentName`, `SkillId`)
- **Unused import** `RecompileAgentsResult` removed from `compile-agents.ts`
