# Release 0.128.0 (2026-04-14)

CLI surface cleanup driven by a top-to-bottom audit of every command. One command removed, three flags removed, and `validate` rewritten from a polymorphic 3-mode design into a single zero-arg command that validates every source, plugin, skill, and agent the CLI knows about.

## Breaking

- **Removed command:** `build stack`. Use `build plugins` or rebuild via `init` with a stack preset. The hybrid interactive/non-interactive model diverged from the rest of the `build *` family; the command is gone.
- **Removed flag:** `compile --agent-source`. Undocumented; threaded an optional remote agent-partials source that nobody used.
- **Removed flag:** `doctor --source`. Ran a source-reachability check that now lives in the config resolution path.
- **Removed flags:** `validate --source`, `validate --plugins`, `validate --all`, and the positional `[path]` argument. `validate` now takes zero arguments.

## Changed

- `validate` rewritten. Running `agentsinc validate` now executes four passes over everything the CLI tracks:
  1. **Registered sources** (primary + extras from `resolveAllSources`). Remote URLs (`github:…`, `https://…`) are skipped with a `— skipped (remote source)` row, since the user isn't the author.
  2. **Installed plugins** — `~/.claude/plugins/` and `<project>/.claude/plugins/`.
  3. **Installed skills** — `~/.claude/skills/` and `<project>/.claude/skills/`. Validates `SKILL.md` + `metadata.yaml` per skill; uses the strict schema or `customMetadataValidationSchema` based on the `custom` flag.
  4. **Installed agents** — `~/.claude/agents/*.md` and `<project>/.claude/agents/*.md`. Validates frontmatter; enforces kebab-case on `name`.
  - All four passes feed a single `Result: N error(s), N warning(s)` summary. Exit non-zero on any error; warnings are non-fatal. Skills and agents passes dedupe when `cwd === $HOME` to match `doctor`.
- `validateSource` extended with three additional phases so marketplace authors running `validate` from inside a source repo get full coverage: stack `config.yaml` + embedded skill metadata, source-side `src/agents/**/metadata.yaml`, and runtime-loaded `config/{skill-categories,skill-rules,stacks}.ts` Zod-checked against their schemas. All three phases run in parallel; absent targets skip silently.
- `doctor` no longer spreads `BaseCommand.baseFlags` (matches its actual behavior — `--source` was never meaningful there).

## Removed

- `validateAllSchemas` and its supporting helpers (≈300 LOC) deleted from `schema-validator.ts`. No production caller remained. The file now exports only `formatZodErrors`.
- `STATUS_MESSAGES.FETCHING_AGENT_PARTIALS` — orphaned after `compile --agent-source` was removed.
- Obsolete `schema-validator.test.ts` — every test targeted deleted symbols.

## Added

- `docs/reference/commands.md` — full surface reference for every command. Command matrix, per-command detail, conventions, known gaps, and a running AI-docs drift log.
- `docs/guides/…`-level README nudge: polyglot-setup hint added to the scope bullet so users jumping between frameworks install domain agents at project scope and keep meta agents global.
- Shared test fixture file `mock-source-files.ts` with canonical valid-file-shape constants for the five published source artifacts.
- `renderAgentMd` helper in `content-generators.ts` for installed-agent `.md` frontmatter (distinct from `renderAgentYaml` which writes source-side metadata).
- D-181 YOLO mode toggle implementation plan at `todo/D-181-yolo-mode-toggle.md`.
