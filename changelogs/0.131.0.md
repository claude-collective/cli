# Release 0.131.0 (2026-04-15)

Three scaffolding commands gated behind feature flags pending deeper lifecycle fixes. Comprehensive TODO coverage for the underlying matrix composition work. No user-facing command removals; disabled commands exit non-zero with a clear message pointing at the tracking tasks.

## Changed

- **`cc new skill` gated behind `FEATURE_FLAGS.NEW_SKILL_COMMAND`** (default `false`). Running it exits non-zero with `"currently disabled while we fix critical issues around the custom-skill lifecycle. See todo/TODO.md (D-212)..."`. The command's `scaffoldSkillFiles` library function stays un-gated â€” `new marketplace` still uses it internally.
- **`cc new agent` gated behind `FEATURE_FLAGS.NEW_AGENT_COMMAND`** (default `false`). Disabled pending D-213 â€” the command requires `agent-summoner` meta-agent to be pre-compiled but gives a misleading `"run compile"` error when missing, and output lands in non-standard `.claude/agents/_custom`.
- **`cc new marketplace` gated behind `FEATURE_FLAGS.NEW_MARKETPLACE_COMMAND`** (default `false`). Disabled pending D-214 â€” runtime matrix composition has ~22 hardening gaps (silent ID overwrites, orphaned custom skills, broken multi-source relationships, schema drift) that affect anyone consuming a scaffolded marketplace.

All three gates follow the identical shape used in prior flag introductions:

```ts
async run(): Promise<void> {
  if (!FEATURE_FLAGS.NEW_SKILL_COMMAND) {
    this.error("...disabled...", { exit: EXIT_CODES.ERROR });
  }
  // ... rest unchanged
}
```
