# Release 0.74.13 (2026-03-15)

## Changed

- Wizard step titles now use a padded block style (`#eee` background) instead of plain colored text
- Skill tags show scope badge (`G`/`P`) with styled background before the skill name
- Category section headers use `#383838` background with white text when focused, gray `*` for required marker
- Source grid: selected but unfocused sources no longer show colored borders
- Agents step: use chevron indicator (`❯`) instead of dot, scope badge styled to match skill tags, description only shown on focus
- Agents step scope toggle (`S`) now shown in footer
- Sources step `L`/`P` hotkeys moved from inline text to footer `DefinitionItem` toggles
- Remove `↑/↓ navigate` from wizard footer, remove active highlight on Labels toggle
- Source choice screen ("Use recommended" vs "Customize") gated behind `SOURCE_CHOICE` feature flag (default: off) — sources step now goes straight to customize view

## Fixed

- Matrix resolver: fix line-length formatting in `computeAdvisoryState` and `getAvailableSkills`

## Test updates

- Update `waitForStableRender` to use `"select"` instead of removed `"navigate"` anchor
- Update 50+ e2e tests to wait for `"Customize skill sources"` instead of `"technologies"` (choice view removed)
- Remove choice view navigation steps (`arrowDown` to customize option) from e2e tests
- Remove choice view unit tests from `step-sources.test.tsx`
- Remove `"navigate"` assertions from integration and e2e tests
