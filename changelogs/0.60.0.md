# Release 0.60.0 (2026-03-07)

## Added

- `SkillSlug` union type (~85 values) replacing `SkillDisplayName` — provides compile-time type checking for relationship rules
- `slug` field (required) on `ExtractedSkillMetadata`, `ResolvedSkill`, and `SkillOption` — kebab-case short key for alias resolution and search
- `displayName` field (required) on `ResolvedSkill` and `SkillOption` — title-cased label for UI display
- `SkillSlugMap` type with bidirectional `slugToId` / `idToSlug` maps on `MergedSkillsMatrix`
- `CompatibilityGroup` type — symmetric compatibility groups replacing per-skill `compatibleWith` arrays
- `SetupPair` type — bidirectional setup relationships replacing per-skill `requiresSetup`/`providesSetupFor`
- `Recommendation` type — flat opinionated picks replacing directional `RecommendRule`
- `SkillDefinitionMap` type alias for `Partial<Record<SkillId, SkillDefinition>>`
- `skipExtraSources` option on `SourceLoadOptions` for faster config-types generation
- `slug` field in `metadata.schema.json` for JSON schema validation
- `new:marketplace` now creates `config.ts`, registers custom schema values, and generates `config-types.ts`

## Changed

- Slug map is now derived from skill metadata at extraction time instead of a hand-maintained aliases map
- Relationship rules use group-based declarations (conflicts, compatibility groups, setup pairs) instead of per-skill arrays
- `ResolvedSkill.recommends: SkillRelation[]` replaced by `isRecommended: boolean` + `recommendedReason?: string`
- `isRecommended()` in matrix-resolver now checks flat list membership + compatibility group overlap
- `info` command displays slug and displayName instead of alias
- `search` command parses frontmatter for real skill data from third-party sources
- `new:skill` template includes `slug` field, no longer updates skill-rules aliases
- Wizard alias resolution uses `slugMap.idToSlug` instead of `displayNames`
- `buildLocalSkillsMap` return type narrowed from `Partial<Record>` to `Record`
- `AgentFiles` type in compiler uses `Pick<CompiledAgentData>` instead of redundant inline type
- `build-step-logic` functions use `Pick<SkillOption>` instead of inline type shapes

## Removed

- `SkillDisplayName` type (replaced by `SkillSlug`)
- `aliases` map from `SkillRulesConfig` and `default-rules.ts` (~85 entries)
- `perSkill` section from `SkillRulesConfig` and `default-rules.ts` (~60 entries)
- `PerSkillRules` type
- `RecommendRule` type (replaced by `Recommendation`)
- `displayNameToId` / `displayNames` maps from `MergedSkillsMatrix` (replaced by `slugMap`)
- `requires`, `compatibleWith`, `conflictsWith` fields from skill metadata schemas
- `api-testing-api-testing` references from all default stacks
- Directory-path duplication in `loadSkillsByIds`
- `PR-REVIEW-NOTES.md`
