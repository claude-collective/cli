# Release 0.133.0 (2026-04-17)

Hardens the plugin-install pipeline across `cc init` and `cc edit`: plugin-intent skills never silently fall back to eject, and marketplace resolution is eager (before any filesystem mutation) to prevent partial-state installs.

## Fixed

- **`cc init` partial-state hazard on plugin marketplace resolution failure**. `init.tsx::handleInstallation` previously ran `copyEjectSkillsStep` before `installPluginsStep`. In `mixed` mode, if the marketplace couldn't be resolved, the plugin step hard-errored AFTER eject skills had been copied to `.claude/skills/`, leaving a half-populated project directory with no `config.ts` to recognise it. On next run, `detectInstallation` returned false and the wizard restarted — silently overwriting the stale copies. Fix: marketplace is now resolved via a new `requireMarketplace` helper BEFORE any filesystem mutation when `pluginSkills.length > 0`. If resolution fails, the hard-error fires on a clean disk. See `.ai-docs/agent-findings/2026-04-17-init-partial-state-on-plugin-hard-error.md`.

- **`cc edit` silently skipped plugin install/uninstall and scope migration when marketplace wasn't resolved**. Plugin-intent skill changes (source !== `"eject"`) were gated behind `context.sourceResult.marketplace && ...`, so operations quietly no-op'd when the marketplace was unavailable rather than hard-erroring. Fix: per-skill plugin intent now drives eager marketplace resolution via `requireMarketplace`; if plugin work is requested and the marketplace fails to resolve, the command hard-errors with a message naming the source and remediation. See `.ai-docs/agent-findings/2026-04-16-silent-plugin-install-skip-on-missing-marketplace.md`.

## Changed

- **`requireMarketplace(sourceResult, purpose)` helper** added to both `init.tsx` and `edit.tsx`. Resolves via `ensureMarketplace`; throws `this.error(...)` with `EXIT_CODES.ERROR` if resolution fails. Helper is duplicated across the two commands because `this.error` binds to the oclif `Command` instance; factoring to a shared module would cost more than it saves for ~15 lines.

- **`init.tsx::installPluginsStep` signature**: now takes a resolved `marketplace: string` from the caller (pre-resolved via `requireMarketplace`) instead of calling `ensureMarketplace` itself and falling back to eject on failure. Removed the `allSkills` parameter — no longer needed since the fallback it fed is gone.

- **`edit.tsx` plugin change flow**: computes `addedPluginSkills` and `removedPluginSkills` early-return-first; only demands marketplace resolution when there's actual plugin work to do. Pure eject edits never touch marketplace resolution.

## Added

- **E2E `e2e/lifecycle/init-plugin-marketplace-fail.e2e.test.ts`**: launches `cc init --source <localSource-without-marketplace.json>`, toggles one source to local to force `installMode === "mixed"`, asserts `EXIT_CODES.ERROR`, output contains `"marketplace could not be resolved"` and the source path, `toHaveNoLocalSkills()` passes (the regression assertion — pre-fix the eject copies were on disk), and the `"Skills copied to:"` success banner is absent.

- **E2E `e2e/interactive/edit-plugin-hard-error.e2e.test.ts`**: guards the `cc edit` hard-error path when the marketplace can't be resolved during a plugin-intent skill operation.

- **E2E `e2e/lifecycle/init-dashboard-edit-plugin-install.e2e.test.ts`**: covers the init → dashboard → edit flow with plugin install semantics.
