# Release 0.85.0 (2026-03-25)

## Added

- **D-132: Skip incompatibility markers in exclusive categories** — radio (max 1) categories like Framework no longer show redundant "(incompatible)" labels on unselected skills. The single-selection constraint already prevents conflicts.
- **D-116: Filter Incompatible toggle deselects skills** — pressing F to enable the filter now also deselects any framework-incompatible skills that were already selected, respecting locked skills. Previously the toggle only hid skills from the grid without removing them from selections.
- **Grid-aware `selectSkill` in E2E infrastructure** — `BuildStep.selectSkill` now parses the grid layout to find a skill's (row, col) position and navigates to it. The previous implementation relied on a `>` cursor marker that doesn't exist in the grid.
- **E2E tests** for both D-132 (exclusive suppression) and D-116 (filter deselection)
- **D-150 TODO** — migrate existing `toggleSkill` E2E calls to the new `selectSkill` for correct grid targeting
- **Agent suggestion** — "Declarative programming: no context required" principle documented in `.ai-docs/agent-suggestions/`

## Changed

- `toggleFilterIncompatible` refactored to thin orchestrator calling pure functions (`findIncompatibleWebSkills`, `removeSkillsFromSelections`)
- `isCompatibleWithSelectedFrameworks` exported from `build-step-logic.ts` as shared predicate (was duplicated)
- Vue and Pinia added to base E2E skills — removed `additionalSkills`/`categories` options from `createE2ESource`
- All inline `createMockSkill`/`createMockMatrix` calls in `build-step-logic.test.ts` extracted to named constants in `mock-skills.ts` and `mock-matrices.ts`

## Fixed

- E2E skill count assertions updated from 7 to 9 after adding vue/pinia to base source
- `relationships.e2e.test.ts` discourages test updated slug (vue now resolves in base source)
