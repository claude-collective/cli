# Release 0.122.0 (2026-04-13)

## Added

- **D-200: Source mode change markers in info panel** — `SkillAgentSummary` now shows `~` prefix with transition label (e.g. "Agents Inc → Eject") when a skill changes install mode between plugin and eject
- **D-191: Dual-scope badges in build step** — Skills toggled from global to project scope now show both `G` and `P` badges side by side, computed from excluded tombstone + active config
- **D-189: Lock icon for globally installed skills** — Skills installed at global scope show a lock icon (🔒) after the display name in the build step
- **D-183: Agent dual-scope badges** — Agents step now renders secondary scope badge when an agent exists in both scopes via scope toggle
- **D-197: Change summary improvements** — `logChangeSummary` uses display names instead of raw skill IDs, adds `[G]`/`[P]` scope labels, and shows G→P scope changes as green `+` additions instead of yellow `~` modifications
- **D-196: Agent toggle guard** — Toast "Global agents cannot be changed from project scope" when pressing Space on a globally-installed agent from project scope
- Exported `ConfigChanges` type and `detectConfigChanges` function from `edit.tsx` for testing
- Test helpers `extractNamedSection` and `extractScopeSections` in config-writer tests

## Fixed

- **D-199: P→G reverse toggle for ejected skills** — Toggling an ejected skill back from project to global (undo) no longer shows "Already exists as ejected skill at global scope" toast when an excluded tombstone exists from the prior G→P toggle
- **D-198: Skill scope corruption** — `buildSkillConfigForId` now prefers project-scoped entry over global when duplicates exist in saved configs; config-writer preserves both global and project entries as a complete installation snapshot
- Config-writer deduplication filter reverted — `generateProjectConfigWithInlinedGlobal` no longer strips global entries that also exist at project scope

## Changed

- `SkillTag` component renders `secondaryScope` badge alongside primary scope badge
- `build-step-logic.ts` computes `secondaryScope` from excluded + active config comparison
- `step-agents.tsx` reads `installedAgentConfigs` from store for dual-badge computation
- Config-writer test assertions updated to expect both scope entries preserved
