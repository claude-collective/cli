# Release 0.130.0 (2026-04-15)

Third round of CLI simplification. `search` is now a zero-flag read-only browse; `validate`, `edit`, `new marketplace`, and `update` each lose one more flag; the orphaned `resolveAgentsSource` / `formatOrigin` helpers are gone. Eight more flags pruned, seven fewer exported symbols.

## Breaking

- **Removed flags:**
  - `search --interactive/-i`, `--category/-c`, `--refresh`, `--json` — `search` is now a required-query read-only catalog browse with zero flags. The interactive Ink UI and multi-select-import side effect are gone; use `import skill` to install.
  - `validate --verbose/-v` — validate always emits the summary table. Diagnostic commands shouldn't have a "hide detail" toggle.
  - `edit --agent-source` — last remaining `--agent-source` flag on any command.
  - `new marketplace --output/-o` — always uses cwd as parent; consistent with `new skill` / `new agent`.
  - `update --no-recompile` — auto-recompile after update is the right default. Users who want finer control can run `cc compile` separately.
- **Positional arg required for `search`** — `search` without a query exits 2 with `Missing 1 required arg`.
- **`--source` inheritance dropped** from `search` and `validate` (overrides `baseFlags` to `{}`). Seven commands now: `doctor`, `build plugins`, `build marketplace`, `new skill`, `import skill`, `search`, `validate`.

## Changed

- **`search` rewritten.** Zero-flag, required query, multi-source table output (primary + every registered extra fetched via `giget`), new Source column. File renamed from `search.tsx` → `search.ts` (no more JSX). `runInteractive`, `showSearchUI`, `importSelectedSkills`, `loadAllSourceSkills`, `toSourcedSkill`, `copySearchedSkillsToLocal` all deleted. Multi-source merge is the only behavior — primary source via `loadSource`, extras via `resolveAllSources` + `fetchFromSource` per extra, merged and printed.
- **`validate` zero-flag, zero-arg.** `setVerbose(flags.verbose)`, the verbose param threading through `validateAllRegistered` / `validatePluginsDirectory` / `validateInstalledSkillsDirectory` / `validateInstalledAgentsDirectory`, and the `verbose` param on `printPluginValidationResult` all removed. Always emits the summary table.
- **Unresolved-slug diagnostic promoted** in `src/cli/lib/matrix/skill-resolution.ts` from `verbose()` to `warn()`. When a skill's `requires` / `conflictsWith` / `discourages` / `alternatives` references a slug that isn't in the same source's catalog, the warning is always visible on stderr instead of hidden behind a removed flag. Scope: source authoring integrity — does not fire on installation state, scope splits, or partial installs.
- **`new marketplace` creates in cwd always.** The `flags.output ? path.resolve(flags.output) : process.cwd()` branch collapses to `process.cwd()`.
- **`update` always recompiles agents.** The ternary `flags["no-recompile"] ? [] : await recompileAfterUpdate(...)` collapses to the unconditional call.

## Removed

- **Orphaned `resolveAgentsSource` helper** and `ResolvedAgentsSource` type in `src/cli/lib/configuration/config.ts`, along with the `"--agent-source flag cannot be empty"` error message. No production callers remained after the `--agent-source` sweep.
- **Dead `formatOrigin` function** and `AgentsSourceOrigin` type in the same file. Zero production callers (only its own tests referenced it).
- **`loadAgentDefs` first `agentSource` parameter** dropped — all three callers (`compile`, `init`, `edit`) passed `undefined`. The helper now takes only the options object.
- **`SkillSearch` Ink component and `use-filtered-results` hook** — orphaned by the `search` simplification (449 LOC total: `skill-search.tsx` + `index.ts` + the hook).
- **Orphaned constants and messages**: `UI_LAYOUT` block in `consts.ts` (`MAX_VISIBLE_RESULTS`, `DESCRIPTION_WIDTH`, `COPIED_MESSAGE_TIMEOUT_MS`, `FALLBACK_MESSAGE_TIMEOUT_MS`), `HOTKEY_COPY_LINK` in `hotkeys.ts`, `SUCCESS_MESSAGES.IMPORT_COMPLETE`.
