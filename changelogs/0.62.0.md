# Release 0.62.0 (2026-03-09)

## Added

- Matrix store (`useMatrixStore`) — centralized Zustand store for read access to merged skills matrix with `setMatrix()`, `getMatrix()`, `getSkill()`, `findSkill()`, and `reset()`
- Canonical skill category registry (`getCanonicalSkillCategories()`) in test helpers — lazy singleton for deriving test skill categories
- Matrix store reset in `vitest.setup.ts` `beforeEach` for automatic test isolation
- 12 new issues (D-78 through D-87) and 2 refactor tasks (R-10, R-11) in TODO tracking

## Changed

- Matrix resolver: remove `matrix` parameter from 9 public functions (`resolveAlias`, `getDependentSkills`, `isDiscouraged`, `getDiscourageReason`, `isRecommended`, `getRecommendReason`, `validateSelection`, `getAvailableSkills`, `getSkillsByCategory`) — all read from store
- Wizard store: remove `matrix`/`categories` params from `populateFromStack()`, `populateFromSkillIds()`, `setAllSourcesPlugin()`, `buildSourceRows()`
- Build-step logic: remove `matrix` param from `buildCategoriesForDomain()`, `getSelectedFrameworks()`, `isCompatibleWithSelectedFrameworks()`
- Config generator: remove `matrix` param from `generateProjectConfigFromSkills()` and `buildStackProperty()`
- Source validator: simplified to use matrix store instead of manual matrix construction
- All 8 CLI commands updated to use store-based API instead of parameter threading
- All wizard components and hooks updated to read from matrix store
- `createMockSkill()` simplified — category derived from registry unless overridden
- `writeTestSkill()` simplified — derives slug/category/author from matrix store via `findSkill()`
- `buildSourceRows()` return shape simplified — removed `displayName` and `alias` fields (UI derives from skillId)
- README simplified to minimal marketplace setup guide

## Removed

- `slug`, `displayName`, `description` fields from `SkillOption` type — redundant with matrix store lookups
- `getSkillAlias()` helper from wizard store — replaced by matrix store lookups
- `getSkillDisplayLabel()` re-export from build-step-logic
