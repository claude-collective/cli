# Release 0.104.0 (2026-04-03)

## Added

- Global skill preselection during project init — when a global installation exists, its skills and agents are pre-selected in the wizard when choosing a stack or starting from scratch
- Safe global config merging — `writeScopedConfigs` now loads existing global config and merges new items into it instead of overwriting, preserving skills/agents installed by prior project inits
- `mergeGlobalConfigs()` in local-installer — unions skills, agents, stack entries, domains, and selectedAgents without removing existing entries
- `globalPreselections` and `globalAgentPreselections` wizard state fields for carrying global config through the stack selection step

## Changed

- Stack selection restores global agent preselections after `selectStack` (which previously wiped `selectedAgents`/`agentConfigs`)
- Scratch mode domain toggles are now additive — only toggles domains not already selected from global preselections
- Global config is only written when there are global-scoped items to write (previously wrote empty global config)
