# Release 0.102.0 (2026-04-02)

## Added

- Terminal size guard in `BaseCommand.init()` — checks terminal dimensions before every command, clears screen and blocks until resized if too small
- `clearTerminal()` protected method on `BaseCommand` — extracted from duplicate local functions in init.tsx and edit.tsx
- Loading spinner for `init` and `edit` commands while skills load
- New `CLI_COLORS` entries: `DIM`, `GRAY_1`, `BLACK`, `TOAST_BG`, `TOAST_FG`, `HOVER_BG`
- TODO D-177: Style skill select with left border only, change category colour

## Changed

- Migrated all `CLI_COLORS` from named ANSI colors to hex values for consistent rendering (D-176)
- Edit command output styled with hex colors — green added, red removed, yellow changed, white bold titles, gray annotations (D-171)
- Wizard skill tag default border uses dimmed text color matching state; footer hotkey backgrounds use `LABEL_BG` to match scope badges
- Toast, agent step, skill search, permission checker — hardcoded colors replaced with `CLI_COLORS` constants
- Wizard resize logic removed — now handled centrally by `BaseCommand`
- Compiler `sanitizeCompiledAgentData()` no longer strips Liquid syntax from content fields (identity, playbook, output, critical sections) — LiquidJS does not re-evaluate template syntax inside variable values, so double-curlies in content are safe (D-175)

## Fixed

- Terminal resize warning now shows immediately before the loading spinner, not only after wizard loads (D-174)
- `new-agent` E2E tests: removed `it.fails` markers for duplicate-directory and `--force` flag tests — bugs are now fixed
- E2E resize test assertions updated for new warning wording
