# Release 0.136.0 (2026-04-17)

Expands E2E lifecycle coverage with 7 new tests and ships companion documentation (coverage gap audit, E2E standard updates, doc-map sync). No production code changes.

## Added

- **7 new E2E lifecycle tests** guarding end-to-end user journeys that were previously under-covered:
  - `e2e/lifecycle/compile-after-scope-change.e2e.test.ts` — compiled agent `.md` files reflect the post-edit stack across scope changes.
  - `e2e/lifecycle/doctor-dual-scope.e2e.test.ts` — `cc doctor` correctly surfaces state across dual-scope installs.
  - `e2e/lifecycle/edit-global-fallback.e2e.test.ts` — `cc edit` from a fresh project with prior global install shows dashboard → edit flow end-to-end.
  - `e2e/lifecycle/init-edit-compile-roundtrip.e2e.test.ts` — init → edit → compile round-trip preserves state across all three commands.
  - `e2e/lifecycle/init-edit-error-guards.e2e.test.ts` — error paths for init/edit misuse (bad flags, missing source, pre-existing partial state).
  - `e2e/lifecycle/source-switching-full-cycle.e2e.test.ts` — switching a skill's source (marketplace ↔ eject) across the full install cycle.
  - `e2e/lifecycle/uninstall-reinit-lifecycle.e2e.test.ts` — `cc uninstall` followed by `cc init` produces a clean install without lingering state.

- **`.ai-docs/e2e-coverage-gaps.md`** — systematic audit of what E2E coverage exists and what's missing, cross-referenced against command paths and user journeys. Maps gaps to proposed tests so future coverage work is planned rather than ad-hoc.

## Changed

- **`.ai-docs/standards/e2e/README.md`** — added three Critical Rules:
  - **State-change verification**: any test that runs a command creating/modifying/removing files or config MUST assert both config AND filesystem state; snapshot before and assert identical after for no-op paths.
  - **Page-object key-press rule**: every key-press method in a page object MUST call `await this.waitForStableRender()` before the press on first interaction after wizard launch or step transition.
  - **Never broaden assertions**: when a strict assertion fails, investigate — don't weaken. Use `// KNOWN GAP:` or `it.fails` for fixture limitations or product bugs respectively.
  
  Also updates `STEP_TEXT` reference: `BUILD_CATEGORY_COUNT` → `BUILD_FOOTER`.

- **`CLAUDE.md`** — minor additions to NEVER/ALWAYS rule blocks based on findings from 0.133.0–0.135.0 work.

- **`.ai-docs/DOCUMENTATION_MAP.md`** — synced to rename `detectProjectInstallation` → `detectInstallation` (the actual exported name), corrected message-count inaccuracies, and reflects documentation added in this cycle.

- **`changelogs/0.134.0.md`** — minor post-commit formatting touch-up.

## Notes

- No production source code changes.
- No unit test changes. All additions are in `e2e/`.
