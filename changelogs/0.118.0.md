# Release 0.118.0 (2026-04-09)

## Changed

- **Test utility restructure: helpers.ts monolith → domain-scoped directories**
  - Split 1,334-line `helpers.ts` into `assertions/` (4 files), `factories/` (8 files), `helpers/` (6 files)
  - Created 18 reusable assertion utilities adopted across 356 call sites
  - Created `expected-values.ts` with canonical `EXPECTED_AGENTS` and `EXPECTED_SKILLS` constants
  - Created E2E assertion helpers (`expectPhaseSuccess`, `expectCleanUninstall`, `expectDualScopeInstallation`)
  - Created E2E agent matchers (`toHaveAgentFrontmatter`, `toHaveAgentDynamicSkills`)
  - Migrated all 90+ test files from barrel imports to direct domain imports
  - Zero backward-compatibility barrel re-exports remaining
  - `helpers/index.ts` slimmed from 991 lines to 40 lines

- **E2E assertion completeness improvements**
  - Added `api-developer` agent checks to lifecycle tests (was only checking `web-developer`)
  - Fixed preloaded vs dynamic skill assertions to match E2E stack definition
  - Adopted `expectPhaseSuccess` in 39 call sites across 15 E2E files
  - Adopted `expectDualScopeInstallation` in 8 call sites across 4 E2E files

## Fixed

- `EditableOptions.stack` type changed to `Partial<Record<...>>` to allow partial stack definitions
- `create-e2e-source.ts` import path updated for helpers restructure
- Fixed `toHaveAgentDynamicSkills` assertions that checked preloaded skills in body (should be `toHaveAgentFrontmatter`)
- Fixed `toHaveAgentFrontmatter` assertions that checked dynamic skills in frontmatter (should be `toHaveAgentDynamicSkills`)
- `re-edit-cycles.e2e.test.ts`: added stack config to `ProjectBuilder.editable()` so compiled agents have preloaded skills
