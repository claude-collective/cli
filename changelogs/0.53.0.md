# Release 0.53.0 (2026-02-28)

## Added

- **Config-types regeneration on skill/agent creation** — `new skill` and `new agent` commands now regenerate `config-types.ts` automatically so newly created entities appear in type unions immediately without requiring a separate `compile` step.
- **Sectioned Custom/Marketplace comments in generated types** — Generated union types in `config-types.ts` now include `// Custom` and `// Marketplace` section comments when custom skills, agents, categories, or domains are present, making it easy to see which values are user-defined vs marketplace-provided.
- **`--domain` flag on `new skill` command** — Optional flag to explicitly set the domain for a new skill. Defaults to `"dummy"` when omitted.
- **`DOMAIN` added to `LOCAL_DEFAULTS` and `IMPORT_DEFAULTS`** — Centralized default domain values (`"dummy"` for local, `"shared"` for imported).

## Changed

- **`domain` is now required on skill metadata** — `ExtractedSkillMetadata.domain`, `rawMetadataSchema`, `skillMetadataLoaderSchema`, and `localRawMetadataSchema` all require an explicit `domain` field. Domain is never inferred from the category prefix.
- **`synthesizeCategory()` requires explicit domain** — The `skillDomain` parameter is now required (was optional with category-prefix fallback).
- **`custom` field moved from `CategoryDefinition` to `ResolvedSkill`** — Custom flag now lives on individual skills and agents rather than on auto-synthesized categories.
- **`buildCategoryEntry()` and `generateSkillCategoriesTs()` accept domain parameter** — Domain is passed explicitly instead of being inferred from `KNOWN_DOMAINS` set (which has been removed from `skill.ts`).
- **`custom` flag propagated on `AgentDefinition`** — Agent metadata `custom: true` is now loaded and passed through to config-types generation for sectioned union rendering.

## Removed

- **`KNOWN_DOMAINS` set in `skill.ts`** — Domain inference from category prefix has been eliminated entirely.
- **`custom` field on `CategoryDefinition`** — No longer used for auto-synthesized categories.
- **Domain inference verbose log in `matrix-health-check.ts`** — Removed the `custom` category verbose log that depended on the removed field.
