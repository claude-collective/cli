# Release 0.60.1 (2026-03-08)

## Changed

- `CategoryDomainMap` type alias extracted in `types/matrix.ts` — replaces 15 inline `Partial<Record<Category, { domain?: Domain }>>` occurrences across wizard store and tests
- `PluginConfigResult` simplified to `Omit<LocalInstallResult, "copiedSkills" | "skillsDir">` — removes 20-line redundant type
- `parseAgentFrontmatter` return type narrowed to `Pick<AgentFrontmatter, "name" | "description"> | null`
- `DashboardProps` simplified to `DashboardData & { onSelect; onCancel }` — removes 4 duplicated fields
- `resolveSetupPairs` return type narrowed to `Pick<ResolvedSkill, "requiresSetup" | "providesSetupFor">`
- `buildBoundSkillOptions` return type changed to `SourceOption[]` (imported from `source-grid.tsx`)
- `TestSkill` derived from `Pick<ExtractedSkillMetadata, ...>` — shared fields stay in sync with production types
- `displayName` now required on all `TestSkill` objects (~45 entries updated)
- `writeTestSkill` options made fully optional — `slug`, `category`, `domain` derived from skill ID via `parseSkillId()`
- Compile command skill count logging simplified — removes intermediate `localTotalCount` variable

## Removed

- `alias` field from `TestSkill` — dead code from pre-slug era
- `aliases` field from `TestMatrix` — dead code from pre-slug era
- `aliases: {}` from `diskRules` in `create-test-source.ts` and `validate.test.ts`
- Dead `MockMatrixConfig.aliases` field and `skillAliases` parameter from `createMockMatrixConfig`
- Redundant `slug` and `category` arguments from 35+ `writeTestSkill` call sites
