# Release 0.95.0 (2026-03-28)

## Added

- Lock badge (`🔒 G`) on globally-installed skills in project-scope edit — visually distinguishes locked global skills from regular project skills (D-159)
- Lock badge (`🔒 [G]`) on globally-installed agents in project-scope edit (D-159)
- Toast feedback when pressing space on a locked global skill: `{Name} is globally installed, modify from ~/` — auto-clears after 2s (D-159)
- Toast feedback when pressing space on a sibling skill in an exclusive category locked by a global install: `Category is locked from global install, modify from ~/` (D-159)
- `UI_SYMBOLS.LOCK`, `UI_MESSAGES.GLOBALLY_INSTALLED`, `UI_MESSAGES.GLOBALLY_LOCKED_CATEGORY` constants in `consts.ts`
- `CategoryOption.locked` field — marks skills that are globally installed and untoggleable in project context
- `onLockedToggleAttempt` and `onLockedCategoryAttempt` callbacks on `CategoryGrid` and `useCategoryGridInput`
- `Toast` component (`wizard/toast.tsx`) — reusable styled message display, replaces `ViewTitle`

## Changed

- Domain descriptions in `CheckboxGrid` now always visible on every row (previously only shown on the focused row)
- `ViewTitle` component renamed to `Toast` (`view-title.tsx` → `toast.tsx`)
- `buildCategoriesForDomain` accepts optional `lockedSkillIds` parameter to populate `locked` field on options
- `useFrameworkFiltering` accepts optional `lockedSkillIds` parameter, threads through to `buildCategoriesForDomain`

## Removed

- `src/cli/utils/yaml.ts` and `src/cli/utils/yaml.test.ts` (superseded)
- `src/cli/components/wizard/view-title.tsx` (replaced by `toast.tsx`)
