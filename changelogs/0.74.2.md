# Release 0.74.2 (2026-03-13)

## Fixed

- **Uninstall plugin filtering**: Uninstall now only removes plugins installed by the CLI (matched against config skills list), preserving third-party plugins like `ralph-loop`.
- **Plugin scope cwd routing**: `claudePluginInstall` and `claudePluginUninstall` now resolve the working directory by scope — user-scoped plugins run from the home directory so Claude CLI writes to `~/.claude/settings.json` correctly.
- **Agent recompiler scope routing**: Agents are now routed by scope during recompilation — global agents write to `~/.claude/agents/`, project agents to the installation agents directory.
- **Config scope splitting**: `splitConfigByScope` correctly splits `selectedAgents` and `domains` between global and project configs instead of duplicating them.
- **Project config-types generation**: Project `config-types.ts` now only includes project-scoped items; global items are already available via `GlobalSkillId`/`GlobalAgentName` imports.
- **Project config name inheritance**: Project config no longer inherits `"global"` as its name from the global config spread.
