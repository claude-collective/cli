# Release 0.75.1 (2026-03-16)

## Changed

- **Meta-framework skill IDs renamed** — `web-framework-{nextjs,remix,nuxt,sveltekit,astro}` renamed to `web-meta-framework-{nextjs,remix,nuxt,sveltekit,astro}` for consistency with their category
- **Next.js Fullstack stack description** — Updated from "React + Hono full-stack" to "Next.js + Hono full-stack"
- **E2E tests** — Updated stale skill IDs (`web-framework-nextjs-app-router`, `web-framework-nextjs-server-actions`) to current `web-meta-framework-nextjs`

## Added

- **`$schema` in generated metadata.yaml (D-108)** — `cc new skill` now includes a `# yaml-language-server: $schema=...custom-metadata.schema.json` reference for IDE validation

## Removed

- **`tags` field from skill metadata (D-103)** — Removed from JSON schemas, TypeScript types, Zod schemas, loaders, resolvers, search matching, info command display, plugin compiler, and all test data. Tags were unused in practice and added noise to metadata.
