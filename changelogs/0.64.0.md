# Release 0.64.0 (2026-03-10)

## Added

- **Scope-aware config splitting (D-76/D-80):** `writeScopedConfigs()` splits config by scope during install — global skills/agents write to `~/.claude-src/config.ts`, project items write to `.claude-src/config.ts` with `import globalConfig` preamble. Project `config-types.ts` imports from global and extends with project-only types.
- **Config writer named variables (D-81):** `generateConfigSource()` now extracts `skills`, `agents`, `stack`, and `domains` as typed `const` declarations above the `export default`. The export default references them as a scannable table of contents. Import line dynamically includes only the types that are used.
- **Project config-types generation (D-76):** `generateProjectConfigTypesSource()` generates a `config-types.ts` that imports `SkillId`, `AgentName`, `Domain` from global and extends them with project-only members. `regenerateConfigTypes()` detects global install and uses the project variant automatically.
- **D-74 refinement doc:** Documents remaining gaps (edit-mode agent scope restoration, help modal S key, edit command scope diff) with phased implementation plan.

## Changed

- **Default scope is now global:** Skills and agents default to `scope: "global"` instead of `"project"`, matching the dual-installation model where global is the primary installation scope.
- **Methodology skill injection removed:** `getDefaultMethodologySkills()` and automatic injection of `DEFAULT_PRESELECTED_SKILLS` removed. Users explicitly select methodology skills like any other skill.
- **ProjectSourceConfig merged into ProjectConfig:** `SourceEntry`, `BrandingConfig`, and 8 config fields moved from the separate `ProjectSourceConfig` type into `ProjectConfig` in `types/config.ts`. The old type becomes a deprecated `Partial<ProjectConfig>` alias.
- **`saveProjectConfig` renamed to `writeProjectSourceConfig`:** Reflects its purpose of writing with proper TypeScript format (`import type` + `satisfies` clause).
- **Scope badges dimmed (D-77):** Skill tag scope badges use `dimColor` instead of bright WARNING/INFO colors, and only show when scope is set.
- **step-agents.tsx:** Uses selector-based store access instead of full store object to prevent unnecessary re-renders.
- **R-12 scope expanded:** Refactor task updated from "enforce store-only matrix access" to "delete the matrix object from consumer code" with full consumer migration map.

## Fixed

- **Edit command crash on removed skills:** `getSkill()` threw when displaying removed skills not in the current matrix. Changed to `findSkill()` with optional chaining fallback.
- **Integration test pollution:** Tests now mock `os.homedir()` to prevent `writeScopedConfigs()` from writing to the real `~/.claude-src/` directory.
- **E2E edit-wizard stability:** Uses local E2E source to avoid `claude plugin install/uninstall` commands that hang when a real `claude` binary is present. Timeout increased to 60s.
- **Config round-trip test:** Updated `writeAndLoad()` helper to strip type annotations from `const` declarations that jiti can't resolve.
