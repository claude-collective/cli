# Release 0.108.0 (2026-04-06)

## Fixed

- **Stale agent file after scope toggle**: when toggling an agent from project to global scope via `cc edit`, the old `.md` file was not removed from the project agents directory. Root cause: `mergeConfigs` used simple name keys for agents (unlike skills which already used compound keys), causing excluded tombstone entries to overwrite active entries during config merge. Additionally, `mergeGlobalConfigs` never pruned agents/skills that were moved to project scope.
- **Tombstone pruning in global config**: `writeScopedConfigs` now processes excluded tombstone entries as removal signals, pruning agents and skills from the global config when they are toggled to project scope during project init.
- **Excluded entries leaking into wizard display**: the agents step and confirm step summary now filter out excluded tombstone entries, preventing duplicate entries and incorrect scope badges.
- **Scope migration pre-check**: `migrateLocalSkillScope` now checks if the source directory exists before attempting migration, with a fallback check for already-migrated skills.

## Changed

- **Confirm step scope-change indicators**: skills and agents whose scope changed during an edit session now display with a `~` prefix and warning color, distinguishing them from new (`+`) and unchanged (`•`) entries.
- **Source grid and init formatting**: cleaned up long JSX lines in source-grid, source-grid tests, and init command for consistent code style.
