# Release 0.87.0 (2026-03-26)

## Changed

- **Declarative two-tier command pattern** (D-155): All 14 commands restructured so `run()` reads like pseudocode calling named private methods, with pure functions at the bottom of each file. No behavior changes — purely structural.
- **12 single-use operations dissolved** back into their respective command files:
  - `update-local-skills.ts` -> `update.tsx`
  - `get-dashboard-data.ts` -> `init.tsx`
  - `generate-skill-diff.ts` -> `diff.ts`
  - `detect-config-changes.ts` + `migrate-plugin-scope.ts` -> `edit.tsx`
  - `eject-project.ts` -> `eject.ts`
  - `uninstall-project.ts` -> `uninstall.tsx`
  - `search-skills.ts` -> `search.tsx`
  - `import-skill.ts` -> `import/skill.ts`
  - `scaffold-skill.ts` -> `new/skill.ts`
  - `scaffold-agent.ts` -> `new/agent.tsx`
- **Shared skill generators** (`generateSkillCategoriesTs`, `generateSkillRulesTs`) moved to `lib/skills/generators.ts` (used by both `new/skill.ts` and `new/marketplace.ts`)
- **Operations layer trimmed** from 28 to 17 files — only genuinely shared operations (2+ callers) remain

## Fixed

- **Global skills missing from compiled agents** after init/edit/update: These commands called `compileAgents()` without discovering skills first, causing `recompileAgents()` to fall back to `discoverAllPluginSkills(projectDir)` which only found project-level plugin skills. Global skills (plugin + local) and project local skills were all missed. Now all three commands call `discoverInstalledSkills()` for the full 4-way skill merge before compilation.
