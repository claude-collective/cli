# Release 0.129.0 (2026-04-15)

Continuation of the CLI surface cleanup from 0.128.0. 13 flags pruned across 6 commands, `build marketplace` now reads identity from `package.json`, `doctor` gains per-check error isolation, `validate` dedup is realpath-based (fixes macOS symlink bug), and the test suite gains shared `setupIsolatedHome` + `writeTestPackageJson` helpers.

## Breaking

- **Removed flags:**
  - `doctor --verbose/-v` — diagnostic commands shouldn't have a "hide info" toggle; details are always emitted.
  - `build plugins --skills-dir/-s` — hardcoded to `src/skills/` (marketplace convention).
  - `build marketplace --name`, `--version`, `--description`, `--owner-name`, `--owner-email` — replaced with `package.json` read at cwd.
  - `new skill --output/-o` — auto-detects marketplace context.
  - `new agent --non-interactive/-n` — redundant with `--purpose` (providing a purpose already skips the prompt).
  - `new agent --refresh/-r` — niche cache-busting on a scaffold command; remove the cache dir manually if needed.
  - `import skill --subdir`, `--refresh` — subdir hardcoded to `skills/`; source fetches use `giget` default caching.
- **`--source` inheritance dropped** from 5 commands that don't consume a skills source: `doctor`, `build plugins`, `build marketplace`, `new skill`, `import skill`. `BaseCommand.baseFlags` is overridden to `{}`.

## Changed

- **`build marketplace` reads marketplace identity from `package.json`** at cwd. Required fields: `name`, `version`, `description`. `author` is optional (warns when missing but continues). The `author` field is parsed flexibly:
  - String `"Name <email>"` → `{ name, email }`
  - String `"Name <email> (url)"` (npm's canonical format) → `{ name, email }` (URL discarded)
  - String `"<email>"` (email only, warns) → `{ name: "", email }`
  - String `"Name"` (name only, warns) → `{ name }`
  - Object `{ name, email?, url? }` (URL discarded) → passed through
- **`doctor` gets per-check error isolation.** Each check runs inside a `safeCheck(kind, fn)` wrapper — a throwing check produces a `status: "fail"` `CheckResult` rather than crashing the whole run. Partial results always surface.
- **`doctor` `CheckKind` discriminator** (`"config" | "skills" | "agents" | "orphans" | "installed" | "source"`) tags every `CheckResult`. `formatTips()` now keys remediation hints off `kind`, not message substring — renaming a message can't silently break a tip.
- **`doctor` check ordering:** source check is reordered so a source-unreachable failure marks `checkSkillsResolved` as skipped rather than running it against an empty matrix (previously produced misleading "all skills missing" reports).
- **`validate` dedup is realpath-based.** `cwd === $HOME` detection now uses `fs.realpathSync` on both sides — fixes macOS where `$HOME` is commonly symlinked and string comparison silently misses the collision. The plugins pass also gets the same dedup (was previously missing, causing double-listing when run from `$HOME`).
- **`source-validator` `validateTsConfig` regression fixed.** The named-exports detection added in 0.128.0 was over-eager and falsely flagged valid `export default { ... }` files as "no default export". The detection is simplified to trust `loadConfig`'s null return (jiti already unwraps `default`).
- **`validateMetadataConventions` split into `checkSnakeCaseKeys` and `checkDisplayNameMatches`.** The combined function forced callers to pass fabricated metadata when they only needed one half. The dead wrapper is deleted now that tests are migrated.
- **`new:marketplace` calls `scaffoldSkillFiles` directly** instead of invoking `new:skill` via `runCommand`. Fixes a runtime bug introduced by the `new:skill --output` removal and aligns author resolution with `new:skill`.
- **`config-loader` schema-failure error uses `formatZodIssues(...)`** instead of `JSON.stringify(zodError)`. Readable errors.
- **`formatZodErrors` renamed to `formatZodIssues`** in `schemas.ts` to eliminate the name collision with `schema-validator.ts`'s different-signature export. All 7 call sites updated.

## Added

- **`isCustomMetadata(raw)` helper** in `schemas.ts`. Extracted from `validate.ts` and `source-validator.ts` where the discriminator was duplicated.
- **`UI_SYMBOLS.CHECK` and `UI_SYMBOLS.CROSS`** in `consts.ts` to replace raw `"\u2713"` / `"\u2717"` in `doctor`.
- **`static examples`** blocks on `new skill` and `new agent` — they were the only commands missing examples.
- **`scaffoldSkillFiles` exported** from `new/skill.ts` so `new:marketplace` can call it directly.
- **Test infrastructure:**
  - `setupIsolatedHome(prefix)` in `src/cli/lib/__tests__/helpers/isolated-home.ts` — 9 unit-test files migrated off boilerplate `process.env.HOME = fakeHome` setup/teardown.
  - `writeTestPackageJson(dir, overrides?)` in `helpers/config-io.ts` — 4 call sites migrated off inline `JSON.stringify({...})`.
  - `readMarketplaceJson(path)` in `e2e/helpers/test-utils.ts` — 3 E2E sites migrated.
  - `VALID_PACKAGE_JSON_FILE` canonical fixture in `mock-data/mock-source-files.ts`.
- **New tests:** author-parsing edge cases × 5, `build marketplace` missing-`package.json` error, `new marketplace` marketplace.json-contains-dummy-skill, `validate` four-header assertion, `doctor` kind-based tip discrimination × 2, D-92 source-field unskip (previously `it.skip` in `config-scope-integrity.e2e.test.ts`).

## Removed

- `validateAllSchemas`-era `validateMetadataConventions` wrapper — tests migrated, no production callers remained.
- Dead `RegistryUpdateResult` type in `new/skill.ts` (return value was discarded at the only call site).
- Dead fixtures: `SINGLE_AGENT_STACK_TEMPLATE`, `MULTI_AGENT_STACK_TEMPLATE`, `PIPELINE_MATRIX`, `PIPELINE_TEST_SKILLS` (zero consumers).

## Fixed

- **Helper bug:** `e2e/helpers/create-e2e-plugin-source.ts` still passed `--name` to `build marketplace`, breaking 13 E2E suites via `beforeAll` cascade. Helper now writes `package.json` via `writeTestPackageJson` before invoking `build marketplace`.
- **`CategoryMap` boundary cast** in `new/skill.ts` now uses `skillCategoriesFileSchema` via `loadConfig` rather than `?? {}` fallbacks on must-exist data.
- **`new agent` `as string` cast removed.** `this.exit()` narrowing gap closed with an explicit `throw new Error("unreachable")`.
- **Unified `--force` descriptions** across `new skill`, `new agent`, `new marketplace`, `import skill` — each reads `"Overwrite existing {noun}"`.
- **Examples format** standardized on the `{ description, command }` object form across `build plugins`, `build marketplace`, `import skill`, and the new examples on `new skill` / `new agent`.
