# Release 0.84.1 (2026-03-25)

## Changed

- **schemas**: Replaced `SKILL_ID_PATTERN` hardcoded regex with generated `SKILL_IDS` array for `skillIdSchema` and `CATEGORIES` for `categoryPathSchema` — eliminates drift when new domains are added (D-146)
- **stacks-loader**: Replaced `SKILL_ID_PATTERN` + `getCustomSkillIds()` with `hasSkill()` from matrix-provider; unknown skill IDs now pass through to the downstream pipeline with a warning instead of being silently filtered out
- **source-validator**: Removed redundant category and skill ID format regex checks — matrix health check handles validation
- **source-switcher**: Removed `SKILL_ID_PATTERN` format check, kept only security checks (null bytes, path traversal)
- **wizard-store**: Replaced stale `BUILT_IN_DOMAINS` (missing `ai`, `infra`, `meta`) with `BUILT_IN_DOMAIN_ORDER` from consts
- **marketplace-generator**: Replaced `CATEGORY_PATTERNS` regex array with `DOMAINS`-based lookup; removed `DOMAIN_CATEGORY_OVERRIDES` (`meta` now renders as `meta`, not `methodology`)

## Added

- `hasSkill(id)` non-throwing existence check in `matrix-provider.ts`
- Agent findings for unnecessary test mocks (matrix-provider, internal utilities)

## Fixed

- AI-domain skills (`ai-orchestration-vercel-ai-sdk`, `ai-provider-anthropic-sdk`, `ai-patterns-tool-use-patterns`) no longer produce "Invalid skill ID" warnings in stack config
- Phantom `security` domain prefix removed from validation (was never a real domain — `shared-security-*` is in `shared`)
- E2E `deselectAll` now covers all 8 built-in domains (was capped at 6)
- E2E retry bumped from 1 to 2 to handle flaky wizard startup

## Removed

- `SKILL_ID_PATTERN` export from `schemas.ts`
- `BUILT_IN_DOMAINS` constant from `wizard-store.ts`
- `CATEGORY_PATTERNS` and `DOMAIN_CATEGORY_OVERRIDES` from `marketplace-generator.ts`
- Legacy `(@vince)` suffix from test directory paths
